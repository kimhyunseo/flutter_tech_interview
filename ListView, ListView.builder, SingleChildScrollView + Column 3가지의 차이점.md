# ListView, ListView.builder, SingleChildScrollView + Column 3가지의 차이점
## 한눈에 보는 비교표

| 구분 | ListView | ListView.builder | SingleChildScrollView + Column |
| --- | --- | --- | --- |
| **생성 시점** | 모든 자식 위젯을 한 번에 생성 | 필요한 자식 위젯만 생성 | 모든 자식 위젯을 한 번에 생성 |
| **메모리 사용량** | 많음 | 적음 | 많음 |
| **성능** | 나쁨 | 좋음 | 나쁨 |
| **사용 사례** | 정적인 짧은 목록 | 동적인 개수의 자식 위젯 | 단순 스크롤 레이아웃 |

## 상세 차이점
### SingleChildScrollView + Column
이 조합은 엄밀히 말하면 '리스트'를 위한 도구라기보다
**'스크롤이 가능한 페이지'** 를 만들기 위한 도구

**특징**: Column 안에 있는 모든 자식 위젯을 앱이 실행될 때 메모리에 한 번에 올림

**장점**: 구현이 직관적이며, 리스트 중간에 다른 종류의 위젯을 넣기 편함

**단점**: 데이터가 1,000개라면 1,000개를 한꺼번에 그리므로 성능 저하(버벅임)가 발생

### ListView
`ListView(children: [...])` 형태로 사용하는 방식

**특징**: SingleChildScrollView + Column과 성능 면에서 거의 동일. 자식들을 한 번에 렌더링

**장점**: 코드가 간결하고 직관적

**단점**: **메모리 비효율성**이 가장 큰 문제. 화면에 보이지 않는 수백, 수천 개의 항목도 메모리에 미리 로드하여 생성

### ListView.builder

**특징**: 사용자가 스크롤 내려서 화면에 보이는 시점이 되어야 항목을 생성하고 화면에서 사라진 항목은 메모리에서 해제하거나 재사용함

**장점**: 수만 개의 아이템을 표시해도 메모리 사용량이 적고 성능이 좋음

**단점**: `itemBuilder`와 `itemCount`를 설정해야 하므로 코드가 아주 조금 더 복잡함, 위젯이 자꾸 파괴됨
