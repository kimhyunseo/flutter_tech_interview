# Flutter에서 상수 클래스란 무엇이며 왜 사용하나요?
## Flutter에서 말하는 ‘상수 클래스’란?

앱 전체에서 공통적으로 사용되는 고정된 값(Text Style, Color, Padding, api 경로, 에셋 경로)을 모아 관리하는 클래스
일반적으로 인스턴스화할 필요가 없으므로 static const 키워드를 사용하여 정의

## 주요 특징

- static: 클래스 인스턴스를 생성하지 않고 클래스 이름으로 바로 접근 가능
- const: 컴파일 타임 상수로 만들어 메모리 효율을 높임
- Private Constructor: AppColors._(); 처럼 프라이빗 생성자를 만들어 외부에서 인스턴스를 생성하지 못하도록 방지

## 상수 클래스와 const의 관계

### const란?
- 컴파일 타임에 값이 확정
- 메모리에 하나만 존재
- 값이 절대 변하지 않음

final 과 차이점 알기

```dart
const int maxCount = 10;
```

### 상수 클래스는?
여러 const 값을 의미 있게 묶은 것

```dart
class AppSizes {
  static const double padding = 16.0;
  static const double radius = 12.0;
}
```

## 왜 사용할까?
###  ① 유지보수의 용이성 
상황: 앱 전체에서 사용하는 메인 테마 색상이 '파란색'에서 '남색'으로 바뀐다면?
이유: 상수 클래스가 없다면 수십 개의 파일에서 색상 코드를 찾아 수정해야 하지만, 상수 클래스를 사용하면 단 한 줄의 코드만 수정해도 앱 전체에 반영

### ② 코드의 가독성과 의도 파악 
상황: 코드에 Color(0xFF0055FF)가 적혀 있는 것보다 AppColors.mainBrand라고 적혀 있는 것이 훨씬 이해하기 쉬움
이유: 'Magic Number'나 'Magic String'(의미를 알 수 없는 숫자나 문자열)을 제거하여 코드의 의미를 명확하게 전달

### ③ 런타임 성능 및 메모리 최적화 
상황: Flutter의 위젯 트리에서 const 위젯은 리빌드 시 재사용
이유: 상수 클래스에 정의된 값을 const 위젯의 인자로 전달하면, 런타임에 매번 새로운 객체를 생성하지 않고 컴파일 타임에 생성된 단일 인스턴스를 재사용하므로 메모리 할당량과 CPU 사용량을 줄일 수 있음

## 정리

상수 클래스는 앱 내에서 반복적으로 사용되는 디자인 수치나 설정값들을 static const로 선언하여 한곳에 모아둔 클래스

이를 사용하는 이유는 첫째, 디자인 변경 시 한 곳만 수정하면 되는 유지보수의 편리함 때문이며,
둘째는 코드에서 의미를 명확히 알 수 없는 숫자나 문자열을 제거해 가독성을 높이기 위해서
마지막으로 Flutter의 특성상 컴파일 타임 상수를 활용함으로써 리빌드 시 불필요한 객체 생성을 막고 메모리 효율을 높일 수 있다는 큰 장점
