# 암시적 애니메이션과 명시적 애니메이션의 차이는 무엇인가요?
## 암시적 애니메이션 (Implicit Animations)
"결과만 알려줄게, 중간은 네가 알아서 그려줘."

특징: Animated로 시작하는 위젯들을 사용합니다. (예: AnimatedContainer, AnimatedOpacity)

작동 방식: 위젯의 특정 속성값(예: 높이, 색상)을 바꾸고 setState()를 호출하면, Flutter가 이전 값과 새 값 사이의 중간 과정을 자동으로 계산해 애니메이션을 만듭니다.

장점: 코드가 매우 짧고 간단합니다. AnimationController 같은 복잡한 설정이 필요 없습니다.

단점: 세밀한 컨트롤이 어렵습니다. (예: 애니메이션 도중 멈추기, 무한 반복하기, 특정 구간에서만 빠르게 움직이기 등)

추천 상황: 버튼 크기 변경, 페이드 인/아웃 등 단순한 상태 변화에 적합합니다.

## 명시적 애니메이션 (Explicit Animations)
"내가 시키는 대로, 정확히 이 타이밍에 움직여."

특징: AnimationController와 Tween을 직접 관리합니다. 보통 Transition으로 끝나는 위젯을 쓰거나 AnimatedBuilder를 활용합니다.

작동 방식: 컨트롤러를 통해 애니메이션의 시작, 정지, 반복, 역재생 등을 직접 명령합니다.

장점: 모든 것을 통제할 수 있습니다. 여러 애니메이션을 엮거나(Chained), 상태 변화와 상관없이 계속 움직이게 할 수 있습니다.

단점: 코드가 길어집니다(Boilerplate). SingleTickerProviderStateMixin 설정과 dispose() 처리 등 신경 쓸 게 많습니다.

추천 상황: 로딩 스피너(무한 회전), 복잡한 캐릭터 움직임, 사용자의 드래그에 반응하는 인터랙션 등에 필수적입니다.

## 정리 
"애니메이션의 **라이프사이클(시작, 정지, 반복)을 직접 제어해야 하는가?**라는 질문에 'Yes'라면 명시적 애니메이션을, 단순히 'A 상태에서 B 상태로 부드럽게 변하기만 하면 되는가?'에 'Yes'라면 암시적 애니메이션을 선택합니다."
