# StatefulWidget 과 StatelessWidget 의 차이는 무엇인가요?
## StatelessWidget
StatelessWidget은
상태(state)가 없는 위젯입니다.
즉,
한 번 생성되면 내부 데이터가 변하지 않으며 UI도 변경되지 않습니다.

특징
상태를 저장하지 않음
UI가 고정됨
rebuild는 부모 위젯이 바뀔 때만 발생
가볍고 성능이 좋음
예시 코드
``` dart
class MyText extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Text("Hello Flutter");
  }
}
```

사용 예
아이콘
텍스트
레이아웃 구조
변하지 않는 UI

특징: 위젯이 생성될 때 전달받은 데이터(속성)만 화면에 보여줍니다. 내부적으로 데이터를 바꿀 수 있는 힘이 없습니다.

비유: 사진 또는 인쇄된 전단지. 한 번 인쇄되면 내용을 바꾸려면 전단지를 버리고 새로 뽑아야(부모 위젯이 재빌드되어야) 합니다.

사용 예시: 아이콘, 텍스트 라벨, 로고 이미지 등 고정된 UI.

## StatefulWidget
StatefulWidget은
상태(state)를 가지는 위젯입니다.
즉,
상태가 변경되면 UI가 다시 빌드됩니다.
특징
상태 데이터를 가짐
setState() 호출 시 UI 업데이트
두 개의 클래스 구조
1️⃣ Widget 클래스
2️⃣ State 클래스

``` dart
class Counter extends StatefulWidget {
  @override
  _CounterState createState() => _CounterState();
}

class _CounterState extends State<Counter> {
  int count = 0;

  void increase() {
    setState(() {
      count++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        Text("$count"),
        ElevatedButton(
          onPressed: increase,
          child: Text("증가"),
        )
      ],
    );
  }
}
```
사용 예
버튼 클릭 카운터
애니메이션
네트워크 데이터
입력 폼

특징: 사용자의 입력이나 데이터 변화에 따라 화면을 실시간으로 업데이트할 수 있습니다. 위젯 자체는 불변이지만, 별도의 State 객체가 데이터를 기억하고 관리합니다.

비유: 칠판 또는 스마트폰 화면. setState()라는 분필을 사용해 내용을 지우고 다시 쓸 수 있습니다.

사용 예시: 체크박스, 텍스트 입력창, 숫자가 올라가는 카운터, 서버에서 받아오는 경기 목록 등.

## 정리

|구분	|StatelessWidget|	StatefulWidget|
|-|-|-|
|상태 변화	|불가능 (Immutable)	|가능 (Mutable)|
|구성	|위젯 클래스 하나	|위젯 클래스 + State 클래스|
|생명주기	|단순 (생성/빌드)	|복잡 (initState, dispose 등)|
|Rebuild 트리거	|부모 위젯의 속성 변경	|setState() 호출 또는 속성 변경|
|성능	|비교적 가벼움	|메모리와 CPU를 더 소모함|

"단순히 '변하냐 안 변하냐'의 차이를 넘어, 성능 최적화 측면이 중요합니다. StatefulWidget은 setState()가 호출될 때마다 build 메서드를 다시 실행하므로 꼭 필요한 경우가 아니라면 StatelessWidget을 사용하는 것이 메모리 효율에 좋습니다. 최근에는 Provider나 Riverpod 같은 상태 관리 도구를 사용해 StatelessWidget만으로도 동적인 UI를 효율적으로 구현하는 방식을 선호합니다."

## 꼬리 질문
- "왜 StatefulWidget은 클래스가 두 개로 나뉘어 있나요?"

답변: 위젯 자체는 매번 다시 생성되는 소모품(불변)이지만, State 객체는 위젯이 바뀌어도 메모리에 유지되면서 데이터를 보존해야 하기 때문입니다.

- "StatelessWidget에서 변수 값을 바꾸면 화면이 바뀔까요?"

답변: 변수 값은 바뀔 수 있지만, Flutter에게 "화면을 다시 그려!"라고 알리는 setState()가 없기 때문에 화면(UI)은 그대로 유지됩니다.

- StatefulWidget이 Widget과 State로 나뉘는 이유는?
→ Widget은 immutable(불변) 이기 때문에 상태를 State 객체에서 관리합니다.
