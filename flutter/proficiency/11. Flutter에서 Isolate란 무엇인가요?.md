# Flutter에서 Isolate란 무엇인가요?
## Flutter에서 Isolate란?
**Isolate는 Dart에서 제공하는 독립적인 실행 단위(스레드와 유사하지만 완전히 분리된 메모리 구조)**
다트는 기본적으로 싱글 스레드(Single-Thread) 언어입니다. 하나의 선(Main Isolate) 위에서 UI도 그리고, 이벤트도 처리하죠. 하지만 아주 무거운 작업(용량이 큰 JSON 파싱, 고화질 이미지 처리 등)을 이 메인 선에서 하면 UI가 버벅이는 'Jank' 현상이 발생합니다.

이때 **"별도의 격리된 방(Isolate)"**을 만들어 그 무거운 일을 던져주는 것이 바로 Isolate

## 특징
- 각 Isolate는 자신만의 메모리 힙을 가진다
- 다른 Isolate와 메모리를 공유하지 않는다
- 신은 Message Passing (SendPort / ReceivePort) 방식으로 한다
- UI는 오직 메인 Isolate에서만 가능

## 핵심 특징: "공유하지 않는 격리성"

독립된 메모리: 일반적인 스레드는 메모리를 공유하지만, Isolate는 이름 그대로 메모리를 공유하지 않습니다. * 상태 공유 불가: 각 Isolate는 자신만의 메모리와 이벤트 루프를 가집니다. 따라서 데이터 전달은 '메모리 주소 공유'가 아니라 '메시지 전달(Message Passing)' 방식으로만 가능합니다.

장점: 메모리 공유로 인해 발생하는 복잡한 문제(Race Condition, 데드락 등)를 원천 차단합니다.

## 2. 어떻게 사용하나요?
compute() 함수: 아주 간단한 작업에 사용합니다. 함수를 던지면 알아서 새로운 Isolate를 만들고 결과를 돌려준 뒤 사라집니다.

Isolate.spawn(): 더 복잡하고 오래 걸리는 작업, 혹은 지속적인 양방향 통신이 필요할 때 직접 생성해서 관리합니다.

## 정리
"Isolate는 Dart에서 병렬 처리를 구현하는 독립적인 작업 단위입니다. 일반적인 스레드와 달리 메모리를 공유하지 않고 각자 독립된 이벤트 루프를 가지는 것이 특징입니다. 이를 통해 UI 스레드의 부하를 줄여 성능을 최적화할 수 있으며, 메모리를 공유하지 않기 때문에 뮤텍스(Mutex) 같은 복잡한 동기화 처리 없이 안전하게 병렬 프로그래밍이 가능하다는 장점이 있습니다."

## 예상 질문
Q1. "메모리를 공유하지 않는다면 데이터는 어떻게 주고받나요?"

답변: ReceivePort와 SendPort를 사용합니다. 한쪽에서 메시지를 던지면(SendPort), 반대쪽에서 기다리고 있다가 받는(ReceivePort) 방식입니다. 이때 데이터는 복사되어 전달됩니다.

Q2. "Async/Await(비동기)와 Isolate(병렬)의 차이는 무엇인가요?"

답변: 비동기는 한 명이 커피를 내리면서(I/O 대기) 주문을 받는(다른 작업) 효율적인 시간 관리라면, Isolate는 아예 주방 인원을 한 명 더 늘리는(멀티 코어 활용) 개념입니다. CPU 연산량이 많은 작업은 Isolate가 필요합니다.

Q3. "Isolate를 생성할 때 발생하는 비용이나 단점은 없나요?"

답변: Isolate는 생성할 때 자신만의 힙(Heap) 메모리를 할당받아야 하므로 **생성 비용(Overhead)**이 꽤 큽니다. 따라서 너무 가벼운 작업에 남발하면 오히려 메인 스레드보다 느려질 수 있습니다.

Q4. "그럼 모든 무거운 작업에 Isolate를 써야 할까요? 기준이 있나요?"

답변: 보통 프레임 드랍이 발생할 정도의 무거운 연산(보통 16ms 안에 처리가 힘든 작업)에 사용합니다. 단순한 네트워크 호출(API 통신)은 다트의 비동기(Future)가 이미 내부적으로 효율적으로 처리하므로 굳이 Isolate를 쓸 필요는 없습니다.
