# Flutter에서 스로틀링은 무엇인가요? 예를 들어 설명하세요.
## 1. 스로틀링(Throttling)의 정의
스로틀링은 짧은 시간 동안 이벤트가 연속해서 발생할 때,
일정한 시간 간격(예: 0.5초) 동안 최대 한 번만
로직이 실행되도록 제한하는 기법

핵심 목적: 무분별한 함수 실행을 막아 성능 최적화 및 불필요한 리소스(API 호출 등) 낭비 방지
작동 방식: 이벤트가 아무리 많이 들어와도, 설정한 "주기"가 지나지 않으면 다음 실행을 무시

## 2. 왜 필요한가?
Flutter 앱에서는 이런 상황이 자주 발생:
버튼 연타
스크롤 이벤트
검색 자동완성
터치/드래그
API 중복 호출

👉 그대로 처리하면?
❌ 서버 과부하
❌ 성능 저하
❌ UI 버벅임
❌ 중복 요청

그래서 빈도 제한이 필요함 → Throttling

## 스로틀링(Throttling)의 단점
### ① 마지막 이벤트의 누락
내용: 정해진 주기(Tick)마다 실행되다 보니, 사용자의 마지막 동작이 주기에 걸리지 않으면 실행되지 않고 무시될 수 있습니다.
예시: 마우스 스크롤 스로틀링을 1초로 설정했는데, 사용자가 1.1초에 스크롤을 딱 멈췄다면? 1초 시점의 데이터만 반영되고 마지막 0.1초의 움직임은 UI에 반영되지 않아 화면이 어색하게 멈출 수 있습니다.

### ② 데이터의 부정확성
내용: 실시간으로 변하는 데이터를 일정한 간격으로만 갱신하므로, 화면에 표시되는 정보가 실제 데이터와 아주 짧은 시간 동안 불일치할 수 있습니다.
