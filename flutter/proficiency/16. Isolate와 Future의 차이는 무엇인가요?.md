# Isolate와 Future의 차이는 무엇인가요?
## 1. Future: "혼자서 하는 시간 관리" (Asynchronous)
Future는 **메인 스레드(Main Isolate)**의 이벤트 루프 안에서 돌아갑니다.

원리: 작업을 시작해두고 결과가 올 때까지 기다리는 동안, 다른 UI 작업을 처리합니다. (Non-blocking)

비유: 요리사가 가스레인지에 물을 올려두고(Future 시작), 물이 끓는 동안 채소를 써는 것과 같습니다. 요리사는 여전히 한 명입니다.

주 용도: 네트워크 요청(API 호출), 데이터베이스 쿼리, 파일 읽기 등 **'기다림'**이 필요한 I/O 작업.

한계: 만약 요리사가 채소를 써는 게 아니라 '냉동 고기 100근을 직접 다지는 작업'을 시작하면, 물이 끓어도 확인할 시간이 없어 주방(UI)이 멈춰버립니다.

## 2. Isolate: "새로운 일꾼 고용" (Parallelism)
Isolate는 메인 스레드와 완전히 분리된 별도의 메모리와 CPU 코어를 사용합니다.

원리: 메인 스레드와 별개로 동시에(Parallel) 실행됩니다.

비유: 고기 다지는 일이 너무 힘들어 다른 주방에 있는 조수를 부르는 것과 같습니다. 조수가 고기를 다지는 동안 나는 내 주방에서 계속 요리를 할 수 있습니다.

주 용도: 복잡한 수학 계산, 대용량 이미지/비디오 프로세싱, 수만 개의 JSON 데이터 파싱 등 'CPU를 많이 쓰는' 작업.

## 3. 핵심 차이 요약

| 구분 | Future (비동기) | Isolate (병렬) |
| :--- | :--- | :--- |
| **작업 단위** | 하나의 스레드 내에서 **'기다림'** | 별도의 스레드에서 **'독립적 실행'** |
| **핵심 원리** | Non-blocking I/O | Parallel Processing |
| **메모리** | 공유 (Shared) | 격리 (Isolated) |
| **적합한 작업** | 네트워크, DB, 파일 읽기 (I/O Bound) | 이미지 처리, 암호화, 계산 (CPU Bound) |
| **성능 향상** | 응답성 유지 (Latency Hiding) | 처리 속도 향상 (Throughput Increase) |
