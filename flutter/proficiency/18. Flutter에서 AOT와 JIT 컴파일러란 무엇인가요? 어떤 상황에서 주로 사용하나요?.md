# Flutter에서 AOT와 JIT 컴파일러란 무엇인가요? 어떤 상황에서 주로 사용하나요?

## 1. JIT (Just-In-Time): "그때그때 바로바로"
JIT는 프로그램이 **실행되는 도중(Runtime)**에 코드를 기계어로 번역하는 방식

특징: * 코드를 수정하고 저장하면 즉시 화면에 반영되는 **'핫 리로드(Hot Reload)'**의 일등 공신

컴파일러가 소스 코드를 실시간으로 해석하여 실행하기 때문에 가상 머신(Dart VM)이 필요

장점: 개발 속도가 비약적으로 빠름 (수정 -> 확인까지 1초 미만)

단점: 실행 중에 번역 작업을 병행하므로 앱의 초기 구동 속도가 느리고 성능 부하가 있습니다.

사용 상황: **디버그 모드(Debug Mode)**에서 개발 및 테스트를 할 때 사용합니다.

``` dart
flutter run
Hot Reload
```

## 2. AOT (Ahead-of-Time): "미리미리 완벽하게"
AOT는 앱을 배포하기 전, **실행 전(Build time)**에 소스 코드를 해당 기기의 네이티브 기계어로 모두 번역해두는 방식입니다.

특징: * 실행 시점에 번역할 필요가 없으므로 앱이 켜지는 속도가 매우 빠릅니다.

이미 기계어로 바뀌어 있으므로 Dart VM이 필요 없으며, 최적화가 완벽히 되어 있습니다.
장점: 최고의 런타임 성능과 부드러운 애니메이션을 보장합니다.
단점: 코드를 수정할 때마다 전체를 다시 빌드해야 하므로 시간이 오래 걸립니다. (핫 리로드 불가)

사용 상황: **릴리즈 모드(Release Mode)**로 앱 스토어에 출시하거나 성능을 최종 점검할 때 사용합니다.

## 구분	
| JIT (Just-In-Time) | AOT (Ahead-of-Time) |
| --- | --- |
| 번역 시점 | 앱 실행 중 (Runtime) | 앱 실행 전 (Build time) |
| 핵심 무기 | 핫 리로드 (Hot Reload) | 네이티브 성능 (Fast Startup) |
| Flutter 모드 | Debug Mode | Release / Profile Mode |
| 비유 | 외국인과 대화하며 동시통역 받기 | 미리 번역된 책을 읽기 |
| 개발 경험 | 매우 쾌적함 | 느리고 답답함 |

``` dart
flutter build apk
flutter build ios
```

## 정리

"Flutter는 개발 효율과 앱 성능을 모두 잡기 위해 JIT와 AOT 컴파일 방식을 모두 사용합니다.
개발 단계인 Debug 모드에서는 JIT를 사용하여 '핫 리로드'를 통한 빠른 피드백을 얻고,
최종 배포 단계인 Release 모드에서는 AOT를 사용하여 코드를 네이티브 기계어로 미리 컴파일함으로써
사용자에게 최적화된 성능과 빠른 실행 속도를 제공합니다."
