# 이벤트 루프란 무엇인가요?
## 이벤트 루프란?
이벤트 루프(Event Loop) 는
앱에서 발생하는 이벤트와 비동기 작업을 순서대로 처리하는 반복 구조

Flutter(Dart)는 싱글 스레드 기반으로 동작하기 때문에
UI가 멈추지 않도록 이벤트 루프가 비동기 작업을 관리합니다.

## Flutter(Dart) 이벤트 루프 구조
Dart에는 두 가지 큐가 있음

| 큐               | 설명                |
| --------------- | ----------------- |
| Microtask Queue | 가장 먼저 실행되는 큐      |
| Event Queue     | 일반 비동기 작업이 들어오는 큐 |

## 1. 마이크로태스크 큐 (Microtask Queue) - "최우선 순위"

성격: 아주 짧고 시급하게 처리해야 하는 내부적인 작업들

특징: 이벤트 큐보다 무조건 먼저 처리. 마이크로태스크 큐가 텅 빌 때까지 이벤트 루프는 다음 단계로 넘어가지 않음

## 2. 이벤트 큐 (Event Queue) - "일반 순위"

성격: 외부에서 발생하는 대부분의 작업들

예시: 사용자의 터치 입력, 파일 읽기 완료, 타이머(Timer), 네트워크 응답(Future) 등


## 작동 프로세스 (Priority)
이벤트 루프는 다음 로직을 무한히 반복합니다:

- 마이크로태스크 큐에 할 일이 있는가? → 있다면 전부 처리할 때까지 실행.
- 마이크로태스크 큐가 비었는가? → 이벤트 큐에서 하나를 꺼내서 처리.
- 다시 1번으로 돌아가 반복.

주의: 만약 마이크로태스크 큐에서 계속해서 새로운 마이크로태스크를 만들어내면, 이벤트 큐에 쌓인 '클릭 이벤트'나 'UI 그리기'가 실행되지 못해 앱이 굳어버립니다.
이를 '이벤트 기아(Event Starvation)' 현상이라고 합니다.

## Flutter에서 중요한 이유
Flutter에서는 이벤트 루프가 UI 렌더링과 비동기 작업을 동시에 관리합니다.
예를 들어
- 네트워크 요청
- 파일 읽기
- 애니메이션
- 사용자 터치 이벤트
이런 것들이 모두 이벤트 루프를 통해 처리됩니다.
그래서
무거운 작업을 메인 이벤트 루프에서 실행하면 UI가 멈출 수 있습니다.

## Flutter에서 해결 방법
그래서 Flutter에서는 무거운 작업을 Isolate로 분리합니다.
- 메인 Isolate → UI 처리
- 다른 Isolate → CPU 작업
예
compute(parseJson, jsonString);

## 꼬리 질문

- "만약 이벤트 루프에서 실행되는 함수 하나가 10초 동안 끝나지 않으면 어떻게 되나요?"

답변: 이벤트 루프가 다음 할 일을 꺼내지 못하고 멈추게 됩니다. 이로 인해 UI 업데이트가 중단되어 화면이 멈추는(Jank/Freeze) 현상이 발생합니다.

- "UI 렌더링(Frame) 작업은 어떤 큐에서 처리되나요?"

답변: 보통 이벤트 큐에서 처리됩니다. 따라서 이벤트 큐에 너무 무거운 작업이 쌓여 있으면 60fps나 120fps의 부드러운 화면 전환이 불가능해집니다.
