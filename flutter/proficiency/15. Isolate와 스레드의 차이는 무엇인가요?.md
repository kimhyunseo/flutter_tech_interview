# Isolate와 스레드의 차이는 무엇인가요?
스레드는 메모리를 공유하지만, Isolate는 메모리를 공유하지 않는다.

## 메모리 구조의 차이
| 구분             | Thread             | Isolate                      |
| -------------- | ------------------ | ---------------------------- |
| 메모리            | 공유 (같은 주소 공간을 사용) | 완전 분리 (독립적인 메모리) |
| 동기화            | 필요 (Lock, Mutex 등) | 필요 없음                        |
| 데이터 전달         | 공유 변수 접근           | Message Passing              |
| Race Condition | 발생 가능              | 구조적으로 방지                     |
| 통신 방식         | 공유 변수 직접 접근      | 메시지 패싱 (SendPort/ReceivePort) |
| 동기화 이슈       | 레이스 컨디션, 데드락 위험 (Lock/Mutex 필요) | 없음. 메모리가 격리되어 동기화 불필요 |
| 자원 소모         | 가벼움 (동일 프로세스 내 생성) | 상대적으로 무거움 (독립된 루프/메모리 필요) |

## 1. 메모리 격리 (Memory Isolation)

스레드: 여러 스레드가 하나의 메모리 공간을 공유합니다. 그래서 한 스레드가 값을 바꾸면 다른 스레드도 즉시 알 수 있지만, 동시에 같은 값을 건드리면 데이터가 꼬이는 '레이스 컨디션'이 발생합니다.

Isolate: 이름(Isolate: 격리하다) 그대로 각자만의 섬에 삽니다. 옆 섬에서 무슨 일이 일어나는지 전혀 모르고, 내 섬의 자원은 나만 건드릴 수 있습니다.

## 2. 통신과 복사 (Communication & Copying)

스레드: 포인터(주소값)만 넘겨주면 아주 큰 데이터도 순식간에 전달합니다.

Isolate: 데이터를 보낼 때 **'복사(Copy)'**해서 보냅니다. 한쪽에서 데이터를 보내고 나서 원본을 수정해도 받은 쪽에는 영향이 없습니다. (단, Dart 2.15 이후 대용량 데이터 전송 시 '객체 전송'을 통한 최적화 기능이 추가되긴 했습니다.)

## 3. 동기화 도구 (Synchronization)

스레드: 데이터가 꼬이는 걸 막기 위해 Lock, Mutex, Semaphore 같은 복잡한 도구를 써야 합니다. 자칫 잘못하면 서로 기다리다 앱이 굳어버리는 '데드락'에 빠집니다.

Isolate: 그런 거 안 키웁니다. 메모리를 안 섞으니 싸울 일도 없고, 락을 걸 필요도 없습니다. 프로그래밍이 훨씬 단순하고 안전해집니다.

---

## 정리
“스레드는 하나의 프로세스 내에서 메모리를 공유하며 동작하기 때문에 동기화 문제가 발생할 수 있습니다. 반면 Dart의 Isolate는 각 실행 단위가 독립적인 메모리 힙을 가지며 공유 메모리가 없고, 메시지 전달 방식으로 통신하기 때문에 구조적으로 안전한 동시성을 제공합니다.
