# Flutter에서 비동기 처리를 하는 방법에 대해 설명해보세요.
## 1. Future: "미래에 올 선물 상자"
Future는 지금 당장은 값이 없지만, 미래의 어느 시점에 데이터나 에러가 담길 상자와 같습니다.

용도: 한 번만 일어나는 작업 (네트워크 요청, 파일 읽기, 로그인을 위한 DB 조회 등)

상태: Uncompleted(작업 중) -> Completed with Value(성공) 또는 Error(실패)

## 2. async / await: "비동기를 동기처럼"
비동기 코드를 마치 순서대로 실행되는 일반 코드처럼 읽기 쉽게 만들어주는 문법 설탕(Syntactic Sugar)입니다.

async: 이 함수는 비동기 함수임을 선언합니다. 항상 Future를 반환해야 합니다.

await: 비동기 작업이 끝날 때까지 기다리되, 그동안 메인 스레드는 다른 일(UI 그리기 등)을 할 수 있게 해줍니다.

```Dart
Future<void> loadInningData() async {
  print("데이터 요청 시작...");
  // 2초 동안 기다리지만, 화면은 멈추지 않음!
  final data = await http.get(Uri.parse('https://api.inning.com/games')); 
  print("데이터 수신 완료: ${data.body}");
}
```

## 3. Stream: "끊이지 않는 데이터의 흐름"
Future가 한 번의 응답이라면, Stream은 여러 번에 걸쳐 연속적으로 들어오는 데이터입니다.

용도: 실시간 경기 스코어 업데이트, 채팅 메시지 수신, 사용자의 위치 변화 추적 등.

비유: 한 번 배달 오는 택배(Future) vs 계속 물이 흐르는 수도꼭지(Stream).

## 4. Isolate (병렬 처리)
CPU 집약적인 작업을
메인 UI 스레드와 분리해서 처리.
final result = await compute(parseJson, jsonString);
👉 JSON 파싱, 이미지 처리 등

## 꼬리 질문
### Future와 Stream 차이?

| Future | Stream |
| --- | --- |
| 한 번 결과 | 여러 번 결과 |
| 단일 응답 | 지속적 데이터 |
| HTTP 요청 | 실시간 데이터 |
