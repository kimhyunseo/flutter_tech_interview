# Flutter에서 무한 스크롤, 당겨서 새로고침 기능을 사용하는 이유는 무엇인가요?
## 무한 스크롤
사용자가 리스트를 아래로 스크롤할 때 추가 데이터를 자동으로 불러오는 기능입니다.

### 예
SNS 피드
쇼핑 상품 목록
뉴스 리스트

### 사용하는 이유
- 사용자 경험(UX) 향상
페이지를 이동하지 않고 계속 콘텐츠를 볼 수 있음

- 네트워크 효율성
처음부터 모든 데이터를 불러오지 않고 필요한 만큼만 로드

- 성능 최적화
대량 데이터를 한 번에 렌더링하지 않음

- 끊김 없는 경험 (Seamless UX)
과거 웹사이트처럼 [1] [2] [3] 버튼을 눌러 페이지를 넘길 필요가 없습니다. 사용자의 탐색 흐름을 끊지 않고 계속해서 정보를 제공하여 앱에 머무르는 시간(Retention)을 늘려줍니다.    

### Flutter에서 구현 방법
주로 사용하는 방식
ListView.builder
ScrollController
예
``` dart
if (scrollController.position.pixels ==
    scrollController.position.maxScrollExtent) {
  loadMoreData();
}
```

## 당겨서 새로고침(Pull to Refresh)
당겨서 새로고침은
리스트를 아래로 끌어당기면 데이터를 다시 불러오는 기능입니다.

### 예
이메일 앱
SNS
뉴스 앱

### 사용하는 이유
- 사용자가 직접 데이터 갱신 가능
- 최신 데이터 반영
- 직관적인 UX

### Flutter에서 구현 방법
Flutter에서는 보통 RefreshIndicator 위젯을 사용합니다.
예
``` dart
RefreshIndicator(
  onRefresh: fetchData,
  child: ListView(...)
)
```

## 정리
무한 스크롤은 사용자가 스크롤할 때 추가 데이터를 자동으로 로드하여 UX를 개선하고 네트워크 사용을 효율적으로 관리하기 위해 사용합니다. 당겨서 새로고침은 사용자가 직접 최신 데이터를 갱신할 수 있도록 하기 위해 사용됩니다.
