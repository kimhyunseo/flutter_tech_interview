# Flutter에서 디바운싱은 무엇인가요? 예를 들어 설명하세요.
## 1. 디바운싱(Debouncing)의 정의
디바운싱은 연이어 발생하는 이벤트를 그룹화하여
가장 마지막 이벤트가 끝나고 특정 시간(예: 0.3초)이 지났을 때
한 번만 로직을 실행하는 기법

핵심 목적: 불필요한 연산을 줄이고, 특히 네트워크 비용(API 호출)을 절감하기 위해 사용

작동 방식: 이벤트가 발생할 때마다 타이머를 초기화(Reset)
설정한 시간 내에 새로운 이벤트가 또 들어오면 이전 타이머를 취소하고 다시 기다림

## 왜 필요한가?
Flutter 앱에서 이런 상황이 자주 발생:
검색창 입력
필터 조건 변경
자동완성
실시간 검색

```
예:
h → he → hel → hell → hello
```

👉 매번 API 호출하면?
❌ 서버 과부하
❌ 성능 저하
❌ 배터리 낭비

→ 그래서 마지막 입력만 처리 → Debounce

## 디바운싱(Debouncing)의 단점
### ① 응답 지연 (Latency)
내용: 설정한 시간(예: 500ms) 동안 사용자가 입력을 멈출 때까지 기다려야 하므로, 결과가 화면에 나타나기까지 반드시 지연 시간이 발생합니다.

UX 영향: 사용자는 "어? 왜 결과가 바로 안 나오지? 앱이 느린가?"라고 느낄 수 있습니다. 특히 네트워크 속도까지 느린 환경이라면 지연 체감은 더 커집니다.

### ② 실시간성 저하

내용: 사용자가 계속해서 입력을 이어가거나 이벤트를 발생시키면, 타이머가 계속 리셋되어 로직이 영원히 실행되지 않을 수 있습니다.

예시: 사용자가 검색어를 아주 길게 타이핑하는 동안에는 중간 결과가 전혀 보이지 않아 답답함을 유발할 수 있습니다.

# 스로틀링 vs 디바운싱
Debounce = 멈추면 실행
Throttle = 일정 시간마다 실행

## 5️⃣ 코드 구조 차이
🔹 Debounce 코드 핵심
_timer?.cancel(); // 이전 취소
_timer = Timer(...); // 다시 대기
👉 계속 초기화

🔹 Throttle 코드 핵심
if (isLocked) return;
isLocked = true;
👉 잠금 구조

# 꼬리 질문
## 그럼 디바운싱 혹은 스로틀링을 이 상황에 써도 되나요?
### 검색창에 스로틀링을 쓴다면?

사용자가 '안녕하세요'를 입력하는 동안 0.5초마다 중간 결과(안녕, 안녕하세)가 계속 서버로 전송됩니다. 여전히 불필요한 API 요청이 발생하므로 디바운싱이 더 적절합니다.

### 스크롤에 디바운싱을 쓴다면?

사용자가 스크롤을 멈출 때까지 화면에 아무런 변화가 없다가, 손을 떼는 순간에만 데이터가 로드됩니다. 사용자 경험이 뚝뚝 끊기게 되므로 중간중간 실행되는 스로틀링이 더 적절합니다.

✅ 답변 → 있다 ✅

## "스로틀링과 디바운싱을 라이브러리 없이 직접 구현한다면 어떻게 하시겠어요?"
답변 핵심: "Timer와 불리언 변수를 활용합니다."

상세 답변: "디바운싱은 이벤트가 발생할 때마다 기존 Timer를 cancel()하고 새로운 타이머를 시작하는 방식으로 구현합니다. 스로틀링은 실행 중인지 확인하는 bool 변수나 Timer의 isActive를 체크하여, 이미 타이머가 돌아가고 있다면 이후 들어오는 이벤트를 return으로 무시하는 로직을 작성하겠습니다."
