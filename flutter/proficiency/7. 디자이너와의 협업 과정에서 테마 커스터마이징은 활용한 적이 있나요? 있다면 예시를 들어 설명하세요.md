# 디자이너와의 협업 과정에서 테마 커스터마이징은 활용한 적이 있나요? 있다면 예시를 들어 설명하세요.
## 1. 컬러 팔레트와 ColorScheme 매핑
디자이너가 정의한 'Brand Primary', 'Success', 'Error' 등의 컬러를 Flutter의 ColorScheme에 매핑합니다.

협업 예시: "이번 'Inning' 앱의 메인 컬러는 야구장의 잔디를 상징하는 초록색(#2E7D32)으로 가되, 다크 모드에서는 눈의 피로를 줄이기 위해 조금 더 채도가 낮은 연두색으로 변경해주세요."라는 요청을 반영합니다.

구현: ThemeData(colorScheme: ColorScheme.fromSeed(seedColor: Color(0xFF2E7D32), brightness: Brightness.light))와 같이 설정하여 앱 전체 위젯이 이 색상을 참조하게 합니다.

## 2. TextTheme을 통한 타이포그래피 표준화
디자이너는 보통 'Heading 1', 'Body Bold', 'Caption' 등 텍스트 스타일의 위계를 정해줍니다. 이를 TextTheme에 미리 정의해두면 개발자는 매번 폰트 크기와 굵기를 지정할 필요가 없습니다.

협업 예시: "제목은 'Pretendard' 폰트에 Bold 24px, 행간 1.2로 맞춰주세요."라는 가이드를 textTheme.headlineLarge에 설정합니다.

이점: 디자이너가 나중에 "제목 폰트 크기를 전체적으로 2px씩 키워주세요"라고 요청할 때, 테마 코드 한 줄만 수정하면 앱 전체에 반영됩니다.

## 3. 공통 컴포넌트 테마 (ElevatedButtonTheme 등)
버튼의 둥글기(BorderRadius), 그림자, 패딩 등 반복되는 UI 요소를 커스터마이징합니다.

협업 예시: "모든 버튼은 모서리가 12px 깎인 형태여야 하고, 눌렸을 때만 살짝 어두워져야 합니다."라는 피드백을 ElevatedButtonThemeData에 적용합니다.

## 4. 더 깊은 커스터마이징: ThemeExtension 활용
표준 Flutter 테마 속성에 없는 우리 앱만의 특별한 디자인 토큰이 있을 때 사용합니다.

구현 예시: "우리 앱의 야구 전광판 스타일의 배경색은 특수한 그라데이션이 들어가야 해요."라는 요청이 있다면, ThemeExtension을 사용하여 Theme.of(context).extension<InningColors>()!.scoreboardGradient처럼 사용할 수 있게 만듭니다.
