# MVVM 패턴이란 무엇인가요?

MVVM(Model–View–ViewModel) 패턴은
UI(View)와 비즈니스 로직(ViewModel), 데이터(Model)를 분리하는 아키텍처 패턴
👉 목적은 코드의 분리, 유지보수성 향상, 테스트 용이성

## MVVM 구성 요소
###  Model (데이터 계층)
👉 앱의 실제 데이터와 비즈니스 규칙 담당
API 응답 데이터
DB 데이터
Entity / DTO
Repository

### View (UI 계층)
👉 화면에 보여주는 역할만 담당
StatelessWidget / StatefulWidget
디자인, 레이아웃
이벤트 전달
❗ 로직 거의 없음


### ViewModel (상태 + 로직 담당)
👉 View와 Model 사이 중재자 역할
상태 관리
비즈니스 로직
API 호출
데이터 가공
