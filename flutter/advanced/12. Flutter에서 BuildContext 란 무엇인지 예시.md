# Flutter에서 BuildContext 란 무엇인지 예시를 들어 설명해보세요.
## 정의
BuildContext는 위젯 트리에서 특정 위젯의 위치를 나타내는 객체입니다.
쉽게 말하면, **“위젯이 위젯 트리에서 어디에 있는지, 부모에게 접근할 수 있는 경로”** 를 알려주는 핸들입니다.
위젯 트리 상의 위치 정보 + 부모 위젯 접근 기능을 제공합니다.

## 특징
각 위젯의 build() 메서드에서 매개변수로 전달됨:
```
Widget build(BuildContext context) {
  return Container();
}
```

UI 트리 내에서 다른 위젯의 위치와 상태를 참조할 수 있음
Provider, Theme, MediaQuery 등 상위 위젯 데이터 접근에 사용
StatefulWidget의 경우, setState()와 연계해 UI 갱신 가능

## 왜 사용할까?
위젯은 독립적으로 존재할 수 있지만, 종종 **트리 상단의 정보(부모의 데이터)**가 필요할 때가 있음 
이때 context를 사용하여 트리 위로 거슬러 올라가 데이터를 찾음

- 테마 정보 가져오기: Theme.of(context)
- 화면 이동: Navigator.of(context)
- 다이얼로그 띄우기: showDialog(context: context, ...)
- 미디어 쿼리(화면 크기): MediaQuery.of(context)
