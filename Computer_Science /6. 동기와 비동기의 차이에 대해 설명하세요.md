# 동기와 비동기의 차이에 대해 설명하세요.
## 동기(Synchronous)란?
작업이 끝날 때까지 다음 작업을 실행하지 않는 방식
“끝날 때까지 기다린다.”

정의: 요청을 보낸 후, 그 요청의 응답이 올 때까지 다음 작업을 하지 않고 대기하는 방식
특징: 설계가 매우 직관적이고 순차적 A가 끝나야 B가 시작
비유: 우체국 줄서기 앞사람의 업무가 끝나야 내 차례가 오며, 내 뒤 사람들은 내가 끝날 때까지 아무것도 못 하고 기다려야 함

## 비동기(Asynchronous)란?
작업을 요청하고 기다리지 않고 다음 작업을 먼저 실행하는 방식
“기다리지 않고 다음으로 간다.”

정의: 요청을 보낸 후, 응답을 기다리지 않고 곧바로 다음 작업을 수행하는 방식
특징: 결과가 언제 올지 모르지만, 기다리는 동안 다른 일을 할 수 있어 효율적 결과는 나중에 '콜백'이나 'Future'를 통해 전달받음
비유: 카페 진동벨. 주문을 마친 후 커피가 나올 때까지 자리에 앉아 책을 보거나 휴대폰을 할 수 있습니다. 커피가 준비되면 진동벨(콜백)이 울림

## 플러터에서 비동기 예시

## 동기 vs 비동기의 주요 차이
| 구분    | 동기    | 비동기     |
| ----- | ----- | ------- |
| 실행 방식 | 순차 실행 | 기다리지 않음 |
| 블로킹   | O     | X       |
| UI 멈춤 | 가능    | 방지 가능   |
| 사용 예  | 단순 계산 | 네트워크 요청 |


동기와 비동기의 가장 큰 차이는 작업의 순서를 보장하느냐와 기다리는 동안 다른 일을 할 수 있느냐에 있습니다.
동기는 요청과 결과가 한 자리에서 일어나는 방식으로, 코드가 순차적으로 실행되어 이해하기 쉽지만 작업이 길어질 경우 시스템이 멈추는 블로킹 현상이 발생합니다.
반면 비동기는 요청을 보낸 뒤 결과에 상관없이 다음 로직을 실행하는 방식입니다. 덕분에 자원을 효율적으로 사용하고 사용자에게 끊김 없는 경험을 제공할 수 있습니다.
