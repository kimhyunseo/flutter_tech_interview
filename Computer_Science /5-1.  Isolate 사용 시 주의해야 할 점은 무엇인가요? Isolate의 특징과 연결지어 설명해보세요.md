#  Isolate 사용 시 주의해야 할 점은 무엇인가요? Isolate의 특징과 연결지어 설명해보세요.
## 1. 메모리 독립성 (Isolated Memory)
Isolate의 가장 큰 특징은 **"메모리를 공유하지 않음"**
각 Isolate는 자신만의 힙(Heap) 공간을 가짐

주의점: 전역 변수나 정적(Static) 변수가 공유되지 않음

상세: 메인 Isolate에서 `A = 10`이라고 바꿨어도, 새로 만든 Isolate에서 A를 조회하면 초기값(예: 0)이 나옴
데이터를 주고받으려면 반드시 **메시지 패싱(Port)**을 거쳐야 함

## 2. 통신 비용 (Communication Overhead)
Isolate 간에 데이터를 보낼 때, Dart는 데이터를 복사하거나 직렬화(Serialization) 과정을 거침

주의점: 너무 빈번하거나 너무 큰 데이터 전송은 오히려 성능을 떨어뜨림

상세: 무거운 계산을 Isolate로 보냈는데, 결과값이 수십 MB라면 데이터를 메인 Isolate로 복사해오는 과정에서 화면이 멈출(Jank) 수 있음
배보다 배꼽이 더 커질 수 있으니 전송 데이터 크기를 최소화해야 함

## 3. UI 접근 불가 (No UI Access)
Flutter의 모든 UI 작업(위젯 그리기, setState, BuildContext 사용 등)은 오직 Main Isolate에서만 가능

주의점: Isolate 내부에서 UI를 직접 조작하려고 하면 에러 발생

상세: Isolate는 오직 데이터 계산이나 파일 처리 같은 '비즈니스 로직'만 수행해야 함
계산이 끝나면 결과값만 메인으로 보내고, UI 갱신은 메인 Isolate가 담당

## 4. 리소스 소모 (Resource Heavy)
Isolate는 OS 스레드와 직접 연결되며 각각 독립된 메모리 루틴을 가짐
이는 일반적인 스레드보다 생성 비용이 비싸다는 뜻

주의점: Isolate를 너무 많이, 자주 생성하지 않기

상세: 아주 짧은 작업을 위해 Isolate를 만드는 것은 낭비입니다. 작업이 끝난 Isolate는 반드시 kill()을 호출해 메모리를 해제해야 하며, 반복적인 작업이 필요하다면 Isolate를 유지하며 재사용하는 'Worker Pool' 방식을 고려해야 합니다.
