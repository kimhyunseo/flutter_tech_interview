# Isolate는 어떤 상황에서 사용할까요?
## Isolate란?
Dart에서 서로 메모리를 공유하지 않고 독립적으로 실행되는 실행 단위 (별도의 메모리 + 이벤트 루프)

✔ 스레드랑 비슷해 보이지만
✔ 메모리 공유 ❌
✔ 메시지로만 통신 ⭕

## Flutter에서 Isolate의 주요 사용 시나리오
Flutter는 초당 60프레임(60fps)을 유지하기 위해 약 16ms마다 화면을 다시 그림
만약 어떤 작업이 16ms보다 오래 걸려 메인 스레드를 점유하면
화면이 뚝뚝 끊기는 'Jank(렉)' 현상이 발생

이때 바로 Isolate가 필요

## 주요 사용 상황 (Use Cases)
### 1️⃣ 대용량 데이터 파싱 (JSON Parsing)
서버에서 수만 줄짜리 거대한 JSON 데이터를 가져와 모델 객체로 변환하는 작업
예시: 수천 개의 야구 경기 전적 데이터를 한꺼번에 리스트로 변환할 때

### 2️⃣ 이미지 및 영상 처리 (Image/Video Processing)
이미지를 리사이징하거나, 필터를 적용하거나, 복잡한 인코딩/디코딩 작업을 수행할 때
예시: 사용자가 찍은 사진을 업로드하기 전 고화질로 압축 가공할 때

### 3️⃣ 복잡한 계산 및 암호화 (Heavy Math/Cryptography)
복잡한 알고리즘 연산, 대용량 파일의 해시값(SHA-256 등) 추출, 암호화/복호화 작업 등이 해당됩니다.
예시: 보안을 위해 로컬 DB에 저장된 데이터를 암호화하여 저장할 때

### 4️⃣ 파일 I/O 작업
일반적인 파일 읽기/쓰기는 비동기(Future)로 처리되지만,
파일 크기가 매우 크고 데이터를 가공하면서 읽어야 한다면 Isolate가 유리
