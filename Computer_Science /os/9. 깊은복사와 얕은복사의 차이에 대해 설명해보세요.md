# 깊은복사와 얕은복사의 차이에 대해 설명해보세요. 
## 얕은 복사 (Shallow Copy)
얕은 복사는 객체를 복사할 때, 최상위 수준의 주소값만 복사하는 방식

특징: 복사본을 만들었지만, 그 내부의 참조 타입 데이터(리스트, 객체 등)는 원본과 동일한 주소를 가리킴

문제점: 복사본 내부의 리스트 값을 수정하면 원본의 리스트 값도 함께 바뀌어 버리는 **의도치 않은 사이드 이펙트(Side Effect)**가 발생할 수 있음

비유: 같은 사물함의 열쇠를 하나 더 만들어서 친구에게 준 상황. 친구가 사물함 안의 물건을 바꾸면 나도 바뀐 물건을 보게 됨


## 깊은 복사 (Deep Copy)
깊은 복사는 객체를 복사할 때, 내부에 중첩된 모든 데이터까지 새로운 메모리 공간에 복제하는 방식

특징: 원본과 복사본이 완전히 독립적인 주소를 가짐

이점: 복사본을 아무리 수정해도 원본 데이터에는 전혀 영향을 주지 않음. 데이터의 무결성을 지키기에 가장 안전한 방법

비유: 사물함과 그 안의 내용물까지 그대로 복제해서 새로운 사물함을 만든 상황. 친구가 자기 사물함 물건을 바꿔도 내 사물함은 그대로

한 줄 정의
얕은 복사: 객체의 참조값(주소) 만 복사
깊은 복사: 객체 내부 데이터까지 새 메모리에 통째로 복사

## 얕은 복사 vs 깊은 복사
| 구분    | 얕은 복사 | 깊은 복사    |
| ----- | ----- | -------- |
| 복사 범위 | 참조값만  | 내부 객체까지  |
| 메모리   | 공유    | 완전 분리    |
| 속도    | 빠름    | 상대적으로 느림 |
| 안정성   | 낮음    | 높음       |
