# 스택, 힙 메모리란 무엇이며 어떤 차이가 있는지 비교해서 설명하세요.
## 스택(Stack) 메모리란?
### 1. 정의
함수 호출과 관련된 데이터가 저장되는 메모리 영역

### 2. 특징
LIFO 구조 (Last In First Out)
함수 호출 시 자동 할당
함수 종료 시 자동 해제
빠름 (속도 매우 빠름 ⚡)
메모리 크기가 컴파일 타임에 결정
크기 제한 있음

### 3. 저장되는 것
지역 변수
매개변수
함수 호출 정보 (return address)

### 스택 오버플로우
스택 메모리는 크기가 제한되어 있기 때문에, 너무 많은 함수가 중첩되거나 큰 데이터를 저장하려고 하면 **스택 오버플로우(Stack Overflow)**가 발생할 수 있습니다.

## 힙(Heap) 메모리란?
### 1. 정의
프로그램 실행 중에 동적으로 할당되는 메모리 영역
크기 제한이 거의 없음

### 2. 특징
개발자가 생성
자동 해제 X (GC가 관리하거나 직접 해제)
런타임에 크기가 결정되며, 스택보다 훨씬 큰 공간을 사용할 수 있음
상대적으로 느림
크기 큼
수명 예측 어려움

### 3. 저장되는 것
클래스 인스턴스
객체
동적 데이터


## 3. 스택 vs 힙 비교
| 구분     | Stack        | Heap       |
| ------ | ------------ | ---------- |
| 저장 데이터 | 지역 변수, 함수 정보 | 객체, 인스턴스   |
| 메모리 할당 | 컴파일 타임           | 런타임         |
| 해제 방식  | 자동           | GC(가비지 컬렉터) 또는 수동   |
| 속도     | 빠름           | 상대적으로 느림   |
| 크기     | 작음           | 큼          |
| 수명     | 함수 종료까지      | 명시적 해제전까지 |

"스택은 함수 내부의 지역 변수처럼 잠시 쓰고 버리는 가벼운 데이터를 위해 존재하며, LIFO 구조를 통해 매우 빠른 속도로 자동 관리됩니다. 반면 힙은 객체나 리스트처럼 크기가 크고 수명이 긴 데이터를 자유롭게 저장하기 위한 공간입니다.

효율적인 메모리 사용을 위해서는 스택을 쓰는 것이 좋지만, 프로그램의 유연성을 위해서는 힙이 필수적입니다. 특히 가비지 컬렉션이 있는 언어(예: Dart)에서는 힙 메모리를 어떻게 효율적으로 관리하느냐가 앱의 성능을 결정하는 중요한 요소가 됩니다."
