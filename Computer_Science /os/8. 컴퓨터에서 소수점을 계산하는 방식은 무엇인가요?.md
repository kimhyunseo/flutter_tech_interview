# 컴퓨터에서 소수점을 계산하는 방식은 무엇인가요?
## 컴퓨터에서 소수점 계산 방식
컴퓨터는 소수를 부동소수점(Floating Point) 방식으로 표현하고 계산

### 왜 문제가 생길까?
컴퓨터는 2진수를 사용

👉 10진수의 0.1은
👉 2진수로 정확히 표현 불가능

그래서 근사값으로 저장됨

## IEEE 754 표준 (부동 소수점)
컴퓨터에서 소수점을 표현하는 국제 표준은 IEEE 754
'부동(Floating)'이라는 말은 소수점이 고정되어 있지 않고 둥둥 떠다니며 위치를 바꾼다는 뜻

메모리를 효율적으로 쓰기 위해 숫자를 다음과 같은 형식으로 나누어 저장합니다:
 
부호 비트 (Sign): 양수인지 음수인지 결정 (0 또는 1)
지수 비트 (Exponent): 소수점의 위치를 결정 (얼마나 큰 숫자인지)
가수 비트 (Mantissa/Fraction): 유효 숫자를 저장 (실제 숫자 데이터)

## 실무에서의 해결 방법
금융 앱이나 정밀한 계산이 필요한 프로젝트에서는 이 오차를 방지하기 위해 다른 방식을 사용

- 정수 연산: 소수점에 100이나 1000을 곱해 정수로 변환하여 계산한 뒤 마지막에 다시 나눕니다. (예: 100원 + 200원 = 300원)

- Decimal 클래스 사용: Dart(Flutter)에서는 decimal 패키지나 BigInt 등을 사용하여 숫자를 2진수가 아닌 10진수 방식 그대로 처리하도록 강제함

- 오차 허용 범위 (epsilon): 두 소수가 같은지 비교할 때 a == b 대신 (a - b).abs() < 0.000001 처럼 아주 작은 차이 이내라면 같다고 간주함
