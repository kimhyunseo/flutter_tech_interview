# 커널이란 무엇인가?
## 1. 정의
운영체제의 핵심(core) 부분으로, 하드웨어를 직접 제어하고 시스템 자원을 관리하는 프로그램

## 2. 주요 역할
### 1️⃣ CPU 관리 (프로세스 관리)
어떤 프로세스가 CPU를 쓸지 결정
스케줄링 수행
컨텍스트 스위칭 처리

### 2️⃣ 메모리 관리
메모리 할당 / 해제
가상 메모리 관리
프로세스 간 메모리 보호

### 3️⃣ 파일 시스템 관리
파일 생성/삭제
디스크 접근 제어

### 4️⃣ 장치(Device) 제어
키보드
마우스
네트워크 카드
디스크
👉 하드웨어 직접 제어는 커널만 가능

## 3. 왜 앱이 직접 하드웨어를 못 건드릴까?
보안 때문

만약 앱이:
메모리 직접 수정
디스크 마음대로 접근
CPU 독점
→ 시스템 붕괴

그래서:
하드웨어 접근은 오직 커널만 가능
앱은 반드시 **시스템 콜(System Call)**을 통해 요청해야 함

## 4. 커널 종류
### 1️⃣ 모놀리식 커널
모든 기능이 커널 안에 있음
빠름
구조 복잡

예:
Linux
Unix

### 2️⃣ 마이크로커널
최소 기능만 커널에 둠
안정성 높음
속도 느릴 수 있음

예:
MINIX

### 3️⃣ 하이브리드 커널
둘을 섞은 구조

예:
Windows
macOS
