# 블로킹과 논블로킹의 차이에 대해 설명하세요.
## 블로킹(Blocking)이란?
어떤 작업이 끝날 때까지 제어권을 돌려주지 않는 상태

“끝날 때까지 아무것도 못 한다.”

정의: 호출된 함수가 자신의 작업을 마칠 때까지 제어권을 계속 가지고 있는 방식    
특징: 호출한 쪽(Caller)은 호출된 함수가 값을 반환할 때까지 아무것도 못 하고 그 자리에서 멈춰(Wait) 있어야 함

비유: 직원에게 직접 물어보기. "저기, 이 물건 재고 있나요?"라고 물어본 뒤, 직원이 창고에 가서 확인하고 돌아올 때까지 그 자리에서 꼼짝 못 하고 서서 기다리는 것과 같음

## 논블로킹(Non-Blocking)이란?
작업이 끝나지 않아도 제어권을 즉시 반환하는 방식

“기다리지 않고 바로 다음으로 간다.”

정의: 호출된 함수가 작업 완료 여부와 상관없이 제어권을 곧바로 다시 호출한 쪽으로 돌려주는 방식
특징: 호출한 쪽은 제어권을 바로 돌려받기 때문에, 요청한 작업이 끝날 때까지 기다리지 않고 자신의 다음 로직을 즉시 수행할 수 있음

비유: 식당의 진동벨. 주문을 하면 직원이 즉시 진동벨을 주고 돌려보냅니다. 나는 자리에 가서 내 일을 할 수 있고, 중간중간 "다 됐나요?" 물어보거나(Polling) 나중에 알림을 받음

## 블로킹 vs 논블로킹
| 구분     | 블로킹     | 논블로킹   |
| ------ | ------- | ------ |
| 제어권 반환 | 작업 끝날 때 | 즉시     |
| 흐름     | 멈춤      | 계속 진행  |
| UI 영향  | 멈출 수 있음 | 유지 가능  |
| 예시     | sleep   | Future |
