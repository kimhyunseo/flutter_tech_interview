# 값 형식과 참조 형식의 차이에 대해 설명하세요.
## 값 형식(Value Type)이란?

정의: 변수가 **실제 데이터(값)**를 직접 가지고 있는 형식
저장 위치: 주로 스택(Stack) 메모리에 저장
특징: 변수를 다른 변수에 할당하면 실제 값이 그대로 복사, 두 변수는 완전히 독립적

### 예시
int, double, bool, enum 등 (언어마다 다르지만 주로 기본 자료형)

```dart
int a = 10;
int b = a; // b는 a의 값을 복사
a = 20;
print(b); // 10 (a와 독립적)
```

### 특징
변수 안에 값 자체가 들어 있음
복사 시 값이 그대로 복제됨
서로 독립적

## 참조 형식(Reference Type)이란?
정의: 변수가 실제 데이터가 아닌, 데이터가 저장된 **메모리의 주소(참조값)**를 가지고 있는 형식
저장 위치: 실제 데이터는 **힙(Heap)**에 저장되고, 그 주소값은 **스택(Stack)**에 저장
특징: 변수를 다른 변수에 할당하면 '주소'가 복사됨, 즉 두 변수가 동일한 데이터를 가리키게 됨
예시: Class의 인스턴스, List, Map, String

### 예시
```dart
List<int> a = [1, 2, 3];
List<int> b = a; // b는 a의 주소를 복사
a.add(4);
print(b); // [1, 2, 3, 4] (a와 동일)
```

### 특징
실제 객체는 Heap에 존재
변수는 주소(참조값)만 저장
복사 시 주소가 복사됨
여러 변수가 같은 객체를 가리킬 수 있음

## 값 형식과 참조 형식의 차이
| 구분     | 값 형식     | 참조 형식 |
| ------ | -------- | ----- |
| 저장 방식  | 값 자체 저장  | 주소 저장 |
| 메모리 위치 | 보통 Stack | Heap  |
| 복사 방식  | 값 복사     | 주소 복사 |
| 독립성    | 서로 독립    | 서로 영향 |
| 변경 영향  | 없음       | 있음    |
