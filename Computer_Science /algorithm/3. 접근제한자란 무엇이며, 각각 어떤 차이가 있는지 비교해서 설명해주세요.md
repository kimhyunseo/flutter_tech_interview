# 접근제한자란 무엇이며, 각각 어떤 차이가 있는지 비교해서 설명해주세요.
클래스의 변수나 메서드에 대한 접근 범위를 제한하는 키워드

## 왜 필요한가?
### 1. 데이터 보호
외부에서 마음대로 값을 바꾸지 못하게 함

### 2. 객체 무결성 유지
의도하지 않은 변경 방지

### 3. 유지보수성 향상
내부 구현 변경해도 외부 영향 최소화

---

## Flutter(Dart)에서의 접근 제한자 (중요!)
public, private 같은 키워드가 따로 없습니다.

### Public (기본)
이름 앞에 아무것도 붙이지 않으면 기본적으로 어디서든 접근 가능합니다.

### Private (_ 언더스코어)
이름 앞에 **밑줄(_)**을 붙이면 해당 라이브러리(파일) 외부에서는 접근할 수 없게 됩니다.

```dart
class UserProfile {
  String nickname;   // Public: 외부에서 수정 가능
  int _socialId;     // Private: 이 파일 밖에서는 절대 안 보임

  UserProfile(this.nickname, this._socialId);
}
```

## 정리 
"접근 제한자는 객체의 정보를 은닉하고 외부와의 인터페이스를 제한하여, 코드의 독립성을 높이고 데이터 오염을 방지하기 위해 사용합니다."
