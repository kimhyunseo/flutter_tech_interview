# 싱글톤 패턴이란 무엇인가요?
## 싱글톤 패턴(Singleton Pattern)이란?
애플리케이션 전체에서 단 하나의 인스턴스만 생성하도록 보장하는 디자인 패턴
객체를 여러 번 생성하지 못하게 막고
하나의 객체를 전역에서 공유해서 사용하도록 만드는 패턴

## 왜 쓰나요?
자원 낭비 방지
공통 상태 관리
전역 접근 가능
객체 생성 비용 절감

## 대표적인 사용 예
데이터베이스 연결 객체
설정(Configuration) 객체
로그 관리 객체
캐시 관리자

## 싱글톤 패턴의 장단점
### 장점
메모리 효율: 인스턴스가 하나라 메모리를 아낌
데이터 공유: 전역에서 데이터 상태 공유 용이
강제성: 실수로 인스턴스를 여러 개 만드는 것 방지

### 단점
유연성 하락: 클래스 간 결합도가 높아짐
테스트 어려움: 전역 상태라 단위 테스트가 까다로움
멀티 스레드 주의: 여러 곳에서 동시에 접근 시 충돌 위험

## 정리
싱글톤은 강력하지만 너무 남용하면 **'전역 변수'**처럼 되어버려 코드의 흐름을 추적하기 힘들게 만듭니다. 그래서 최근에는 싱글톤을 직접 구현하기보다, GetIt이나 Riverpod, Provider 같은 의존성 주입(DI) 도구를 사용해 객체의 생명주기를 관리하는 방식을 더 권장하기도 합니다.
