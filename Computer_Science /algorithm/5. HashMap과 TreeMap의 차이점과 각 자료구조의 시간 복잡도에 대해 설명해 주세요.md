# HashMap과 TreeMap의 차이점과 각 자료구조의 시간 복잡도에 대해 설명해 주세요.

데이터를 키-값(Key-Value) 쌍으로 저장할 때 가장 많이 고민하게 되는 두 자료구조, HashMap과 TreeMap을 비교해 드릴게요.

이 둘의 차이는 한마디로 **"속도에 올인할 것인가(Hash), 아니면 정렬된 순서가 중요한가(Tree)"**의 차이입니다.

## HashMap vs TreeMap 차이

### 1. HashMap: "빛보다 빠른 탐색"
해시 테이블(Hash Table)을 기반으로 구현된 자료구조입니다.

작동 원리: 키(Key)를 해시 함수에 넣어 고유한 주소값(Hash)을 만들고, 그 주소에 값을 저장합니다.

특징: * 순서를 보장하지 않습니다. 데이터를 넣은 순서대로 나오지 않고 제멋대로 섞여 보입니다.

null 키를 허용하는 경우가 많습니다 (언어마다 다를 수 있음).

비유: 번호표가 붙어 있는 거대한 사물함. 번호만 알면 바로 열 수 있습니다.

### 2. TreeMap: "질서 정연한 데이터"
이진 탐색 트리(정확히는 Red-Black Tree)를 기반으로 구현되었습니다.

작동 원리: 키를 기준으로 데이터를 나무 가지처럼 뻗어 나가는 구조로 저장합니다.

특징:

키(Key)를 기준으로 자동 정렬합니다. (기본은 오름차순)

정렬을 해야 하므로 null 키를 허용하지 않는 경우가 많습니다 (비교 대상이 없기 때문).

비유: 가나다순으로 꽂혀 있는 도서관의 책장.

## 시간 복잡도

기능 | HashMap (평균) | TreeMap
---|---|---
삽입 (Insertion) | O(1) | O(logn)
삭제 (Deletion) | O(1) | O(logn)
탐색 (Search) | O(1) | O(logn)

HashMap은 해시 충돌이 발생하지 않는 한 **상수 시간 O(1)**이라는 압도적인 속도를 자랑합니다.

TreeMap은 데이터를 넣거나 찾을 때마다 트리를 타고 내려가며 비교해야 하므로 **로그 시간 O(logn)**이 소요됩니다.

## 결론
속도 최우선 + 정렬 필요 없음 → HashMap
정렬 필요 + 범위 기반 처리 → TreeMap

## 1️⃣ HashMap을 쓰는 경우 (현업에서 훨씬 많이 씀)
### 1. 단순 조회가 핵심일 때
예:
로그인 시 사용자 ID → 사용자 정보 조회
상품 ID → 상품 객체 조회
캐시 시스템
Map<String, User> userMap = new HashMap<>();
👉 키로 바로 찾으면 끝.
👉 정렬 필요 없음.
👉 평균 O(1) → 매우 빠름.

### 2. 데이터 순서가 중요하지 않을 때
예:
설정값 저장
API 응답 파싱 후 매핑
JSON → 객체 매핑
👉 순서 의미 없으면 HashMap이 무조건 유리.

### 3. 성능이 가장 중요할 때
대규모 트래픽 환경에서:
초당 수천~수만 건 조회
캐시 레이어
👉 TreeMap 쓰면 log n 누적되어 부담됨.

## 2️⃣ TreeMap을 쓰는 경우 (정렬이 필요한 비즈니스 로직)
### 1. 자동 정렬이 필요할 때
예:
점수 순 정렬
날짜 순 정렬
이름 사전순 정렬
Map<Integer, String> scoreMap = new TreeMap<>();
👉 넣는 순간 정렬 유지
👉 따로 sort() 필요 없음

### 2. 범위 검색이 필요할 때 (이게 핵심)
이건 HashMap으로 거의 불가능.
예:
100점 이상 학생 조회
특정 날짜 범위 조회
가격 10,000원 ~ 20,000원 사이 상품
TreeMap은 이런 메서드 지원함:
subMap()
headMap()
tailMap()
👉 이게 실무에서 TreeMap 쓰는 가장 큰 이유.

### 3. 가장 작은 값 / 가장 큰 값이 필요할 때
firstKey()
lastKey()
예:
최근 날짜
최고 점수
최소 가격

| 상황            | 선택      |
| ------------- | ------- |
| 빠른 조회가 가장 중요  | HashMap |
| 데이터 정렬 필요     | TreeMap |
| 범위 기반 검색      | TreeMap |
| 순서 필요 없음      | HashMap |
| 최소/최대 값 자주 사용 | TreeMap |
| 대용량 캐시        | HashMap |
