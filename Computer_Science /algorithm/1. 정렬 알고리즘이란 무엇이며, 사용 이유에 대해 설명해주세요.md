# 정렬 알고리즘이란 무엇이며, 사용 이유에 대해 설명해주세요.
## 정렬 알고리즘이란?
데이터를 **특정 기준(오름차순, 내림차순 등)**에 따라 재배열하는 알고리즘

예를 들어,
[5, 1, 8, 3] → [1, 3, 5, 8]

숫자뿐 아니라
문자열
객체 (예: 나이순, 점수순)
날짜 데이터
등에도 적용

## 정렬 알고리즘을 사용하는 이유
"왜 굳이 에너지를 써서 정렬을 해야 할까?"에 대한 답은 효율성에 있음

- 탐색 속도의 극대화: 데이터가 정렬되어 있지 않으면 특정 값을 찾기 위해 처음부터 끝까지 다 뒤져야 하지만 (O(n)), 정렬되어 있다면 **이진 탐색(Binary Search)**을 통해 아주 빠르게(O(logn)) 찾을 수 있음. 도서관의 책들이 가나다순으로 정렬되어 있지 않다고 상상해 보세요.

- 중복 데이터 제거: 데이터를 정렬하면 동일한 값들이 서로 이웃하게 되므로, 중복된 항목을 찾거나 제거하기가 훨씬 수월해짐

- 가독성과 분석: 사람이나 시스템이 데이터를 파악할 때, 정렬된 상태는 패턴을 파악하거나 최댓값/최솟값을 찾는 등의 통계적 분석을 훨씬 용이하게 함

- 다른 알고리즘의 기초: 많은 복잡한 알고리즘(크루스칼 알고리즘, 그리디 알고리즘 등)이 사전에 데이터가 정렬되어 있음을 전제로 작동

## 주요 정렬 알고리즘 종류
### 단순하지만 느린 알고리즘 (O(n^2))
데이터 양이 적을 때 사용하기 좋고 구현이 간단합니다.

- 버블 정렬 (Bubble Sort): 인접한 두 원소를 비교하며 큰 값을 뒤로 보냅니다.
- 선택 정렬 (Selection Sort): 가장 작은 값을 찾아 맨 앞과 교체합니다.
- 삽입 정렬 (Insertion Sort): 데이터를 하나씩 뽑아 적절한 위치에 삽입합니다.

### 복잡하지만 빠른 알고리즘 (O(nlogn))
대량의 데이터를 처리할 때 필수적입니다.
- 퀵 정렬 (Quick Sort): 기준점(Pivot)을 정해 데이터를 두 그룹으로 나누며 정렬합니다. (평균적으로 가장 빠름)
- 병합 정렬 (Merge Sort): 데이터를 반으로 쪼갠 뒤 다시 합치면서 정렬합니다. (안정성 보장)
- 힙 정렬 (Heap Sort): 힙 트리 구조를 이용하여 최대/최소값을 뽑아냅니다.
