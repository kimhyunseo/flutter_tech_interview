# 람다식(Lambda Expression)이 무엇인지 설명해주세요.

이름이 없는 함수(익명 함수)를 간결하게 표현한 문법

일반적인 함수와 달리 return 키워드나 중괄호{}를 생략하고 화살표(=>)를 사용하는 경우가 많아 **화살표 함수(Arrow Function)**라고도 불림

(매개변수) => 표현식

- 매개변수: 함수에 전달할 값
- 화살표(=>): "이 매개변수를 가지고 저 표현식을 실행해!"라는 의미
- 표현식: 실행될 코드 한 줄 (자동으로 결과가 반환됨)

## 왜 쓰나요?
- 간결함: 불필요한 코드가 줄어들어 가독성이 좋아짐
- 생산성: 함수를 정의하고 호출하는 과정을 생략하고, 필요한 곳에서 즉석으로 만들어 쓸 수 있음
- 함수형 프로그래밍: 데이터를 가공할 때(필터링, 매핑 등) 리스트의 각 요소에 대해 수행할 동작을 전달하기 매우 편리함

## 람다식(Lambda Expression)의 단점
- 가독성 저하: 너무 복잡한 로직을 람다식으로 표현하면 오히려 코드를 이해하기 어려워질 수 있음
- 디버깅 어려움: 익명 함수는 이름이 없어 디버깅 시 추적하기가 더 까다로울 수 있음
- 재사용성 제한: 이름이 없기 때문에 다른 곳에서 재사용하기 어려움

## 정리

"람다식은 이름 없이 정의되는 익명 함수를 의미합니다. 함수를 별도로 선언하지 않고 변수에 대입하거나 다른 함수의 인자로 직접 전달할 수 있어 코드가 간결해지는 장점이 있습니다. 특히 Flutter에서는 버튼의 콜백 함수나 리스트 데이터를 필터링하고 변환할 때 매우 유용하게 활용됩니다."

## 추가 질문

### 1. "익명 함수(Anonymous Function)와 람다식은 같은 건가요? 차이가 있다면 무엇인가요?"

의도: 용어의 정확한 정의와 Dart에서의 표현 차이를 아는지 묻습니다.

핵심 답변: 사실상 같은 개념을 지칭하지만, Dart에서는 보통 중괄호{}를 사용하는 형태를 익명 함수, 화살표=>를 사용하여 한 줄로 표현하는 것을 **람다식(또는 화살표 함수)**이라 부릅니다. 람다는 단일 표현식만 담을 수 있다는 제약이 있습니다.

### 2. "람다식을 남발했을 때 발생할 수 있는 단점은 무엇일까요?" (실무)

의도: 가독성과 유지보수 측면에서의 균형 감각을 확인합니다.

핵심 답변: 
- 디버깅 어려움: 이름이 없어서 스택 트레이스(Stack Trace)에 의미 있는 이름이 남지 않습니다.
- 가독성 저해: 로직이 복잡한데 람다로 억지로 줄여놓으면 코드를 읽기 힘들어집니다.
- 재사용성 하락: 다른 곳에서 똑같은 로직이 필요해도 재사용할 수 없어 중복 코드가 발생합니다.

### 3. "Dart의 List 메서드 중 람다와 찰떡궁합인 고차 함수(map, where, reduce)의 차이를 설명해 보세요." (실전)

의도: 실무 데이터 가공 능력을 확인합니다.

핵심 답변: 
- map: 요소를 1:1로 변환할 때 사용.
- where: 조건에 맞는 요소만 걸러낼 때(필터링) 사용.
- reduce/fold: 리스트 전체를 하나의 결과값으로 응축(예: 합계)할 때 사용.
