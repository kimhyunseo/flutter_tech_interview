# OS가 메모리 관리를 해야하는 이유에 대해 설명해주시고, 메모리 관리를 위해 어떤 전략을 사용하는지 설명해주세요.

## OS가 메모리 관리를 해야하는 이유
### 한정된 자원의 효율적 배분   
우리가 사용하는 RAM은 용량이 정해져 있음
하지만 우리는 RAM 용량보다 훨씬 큰 게임이나 여러 개의 앱을 동시에 띄우고 싶어 함
OS는 현재 실행 중인 프로그램에만 메모리를 주고, 안 쓰는 건 회수하는 방식으로 가성비를 극대화함

### 격리 및 보호 (Protection & Isolation)
프로세스 A가 프로세스 B의 메모리 영역을 마음대로 읽거나 수정하면 보안과 안정성에 치명적
OS는 각 프로그램에 독립된 가상 공간을 부여하여 서로 침범하지 못하게 성벽을 쌓음

OS는 한정된 메모리를 여러 프로세스가 안전하고 효율적으로 사용할 수 있도록 관리해야 함

## 메모리 관리 전략 (Strategy)
### 가상 메모리 (Virtual Memory)
각 프로세스는
자기만의 0번지부터 시작하는 메모리를 가진 것처럼 보임
하지만 실제 RAM은 공유됨

👉 OS가 주소를 변환해줌

장점
메모리 보호
실제 RAM보다 큰 프로그램 실행 가능
효율적인 관리

### 페이징 (Paging)
메모리를 고정 크기 블록(Page) 단위로 나눔
가상 메모리 → Page
물리 메모리 → Frame
OS가 페이지 테이블로 매핑함
👉 외부 단편화 해결

### 세그먼테이션 (Segmentation)
메모리를:
코드 영역
데이터 영역
스택 영역
처럼 논리적으로 나눔.
👉 의미 단위 관리 가능
하지만 외부 단편화 발생 가능

### 스와핑 (Swapping)
RAM 부족하면:
덜 쓰는 프로세스를 디스크로 내보냄
필요할 때 다시 RAM으로 가져옴
👉 SSD를 임시 RAM처럼 사용

### 페이지 교체 알고리즘 (Page Replacement Algorithm)
RAM이 꽉 찼을 때
어떤 페이지를 내보낼지 결정하는 규칙

종류:
FIFO (선입선출)
LRU (최근 사용 최소)
LFU (최빈 사용 최소)
