# 전역 변수와 정적 변수의 차이는?
전역 변수는 “어디서든 접근 가능한 변수”
정적 변수는 “프로그램 전체 생명주기를 가지는 변수”

## 전역 변수 (Global Variable)
✔ 정의
함수 밖에서 선언된 변수
프로그램 전체에서 접근 가능

✔ 특징
프로그램 시작 시 생성
프로그램 종료 시 해제
어디서든 접근 가능

## 정적 변수 (Static Variable)
✔ 정의
static 키워드로 선언
프로그램 종료까지 유지되는 변수

✔ 특징
수명이 프로그램 전체
선언 위치에 따라 의미가 달라짐

### 함수 내부 static 변수
```dart
void func() {
    static int count = 0;
}
```
함수 안에 있지만
호출이 끝나도 값이 사라지지 않음
다음 호출 시 이전 값 유지
👉 이건 전역 변수랑 다름 (접근은 함수 내부에서만 가능)

### 전역 static 변수
```dart
static int count = 0;
```
파일 내부에서만 접근 가능
다른 파일에서는 접근 불가
👉 접근 범위를 제한한 전역 변수

## 3. 핵심 차이 요약
| 구분 | 전역 변수     | 정적 변수         |
| -- | --------- | ------------- |
| 기준 | 접근 범위     | 생명 주기         |
| 수명 | 프로그램 전체   | 프로그램 전체       |
| 접근 | 어디서든 가능   | 선언 위치에 따라 다름  |
| 목적 | 전역 데이터 공유 | 값 유지 또는 범위 제한 |
| 사용 사례  | 전역 설정값          | 상태 유지, 카운터    |

## 왜 나눠서 쓸까?
> 접근 범위를 통제하고, 의도치 않은 변경을 막기 위해서

### 왜 전역 변수를 그냥 다 쓰면 안 될까?
전역 변수는:
어디서든 접근 가능
어디서든 수정 가능

👉 문제점
누가 값을 바꿨는지 추적 어려움
의존성 증가
테스트 어려움
유지보수 지옥
규모 커지면 거의 재앙
그래서 정적 변수로 나누는 이유
정적 변수는 상황에 따라 다르게 쓰여.

#### 함수 내부 static
```dart
void func() {
    static int count = 0;
}
```
👉 함수 밖에서는 못 건드림
👉 하지만 값은 유지됨

이게 왜 필요하냐면:
값은 유지해야 하는데
외부에서 수정되면 안 될 때

#### 파일 내부 static (C 기준)
```dart
static int count = 0;
```
👉 다른 파일에서는 접근 못 함
👉 해당 파일 안에서만 사용
즉,
“이건 내부 구현이니까 밖에서 건드리지 마세요.”

### 결론
전역 변수는 어디서든 접근 가능하지만, 그만큼 의존성이 커지고 유지보수가 어려워질 수 있습니다.
반면 정적 변수는 수명은 유지하면서도 접근 범위를 제한할 수 있어 안정성을 높입니다.
따라서 두 개념을 나누는 이유는 접근 제어와 코드 안정성을 확보하기 위함입니다.
