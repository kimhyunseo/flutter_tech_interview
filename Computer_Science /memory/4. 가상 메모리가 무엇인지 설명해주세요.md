# 가상 메모리가 무엇인지 설명해주세요.
가상 메모리는 각 프로세스가 실제 물리 메모리와 상관없이
독립적인 메모리 공간을 사용하는 것처럼 보이게 하는 기술입니다.

## 왜 필요한가?
RAM은 한정적임
여러 프로세스가 동시에 실행됨
서로 메모리 침범하면 안 됨
RAM보다 큰 프로그램도 실행해야 함
-> 이걸 해결하기 위해 나온 게 가상 메모리

## 어떻게 동작하나?
### 🧠 핵심 개념: 주소 변환
프로세스는 이렇게 생각해:
> "나는 0번지부터 시작하는 메모리를 가지고 있어."
하지만 실제로는
RAM이 공유되고 있음
그래서 OS가 중간에서
가상 주소 → 물리 주소로 변환해줌

### 핵심 개념: 페이징(Paging)
가상 메모리를 효율적으로 관리하기 위해 메모리를 일정한 크기로 쪼갬
이 단위를 **페이지(Page)**라고 부름

**페이지 폴트 (Page Fault)**: 실행하려는 데이터가 RAM에 없고 하드디스크(가상 메모리)에 있을 때 발생
이때 OS는 하드디스크에서 해당 데이터를 찾아 RAM으로 옮겨옴

**스와핑 (Swapping)**: RAM이 꽉 차면 당장 안 쓰는 페이지를 하드디스크(SSD)로 보내고(Swap-out), 필요한 페이지를 다시 가져오는(Swap-in) 과정

## 왜 가상 메모리를 쓰나요?
### 메모리 부족 해결
실제 RAM 용량보다 큰 프로그램을 실행할 수 있음

### 프로세스 격리 (보안)
각 프로그램이 독립된 가상 공간을 쓰기 때문에, 한 프로그램이 고장 나도 다른 프로그램이나 운영체제의 영역을 침범할 수 없음

### 효율성
앱의 모든 기능을 한꺼번에 쓰지 않기 때문에, 지금 필요한 부분만 RAM에 올려서 자원을 아낄 수 있음
