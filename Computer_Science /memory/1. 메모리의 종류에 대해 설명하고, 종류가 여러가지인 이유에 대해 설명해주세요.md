# 메모리의 종류에 대해 설명하고, 종류가 여러가지인 이유에 대해 설명해주세요.
## 메모리 계층 구조
🔥 빠를수록 비싸고 작다
🧊 느릴수록 싸고 크다

### 레지스터 (Register)
📍 위치: CPU 내부
⚡ 속도: 가장 빠름
📦 용량: 매우 작음 (수십~수백 바이트 수준)
🎯 역할: 지금 당장 연산할 데이터 저장

👉 CPU가 직접 접근하는 공간

### 캐시 메모리 (Cache - L1, L2, L3)
📍 위치: CPU 내부 또는 근처
⚡ 속도: 매우 빠름 (RAM보다 빠름)
📦 용량: 수 MB 수준
🧠 SRAM 사용
🎯 역할: 자주 사용하는 데이터를 임시 저장

#### L1, L2, L3 캐시

| 단계 | 속도       | 크기    | 위치    |
| -- | -------- | ----- | ----- |
| L1 | 가장 빠름    | 가장 작음 | 코어 내부 |
| L2 | 중간       | 중간    | 코어 내부 |
| L3 | 상대적으로 느림 | 가장 큼  | 코어 공유 |

CPU와 RAM 사이의 병목을 줄이기 위해 존재

### 주기억장치 (RAM, Main Memory)
📍 위치: 메인보드
⚡ 속도: 캐시보다 느림
📦 용량: GB 단위
🧠 DRAM 사용
❗ 휘발성 (전원 꺼지면 사라짐)
🎯 실행 중인 프로그램과 데이터 저장

앱 실행 시:
SSD → RAM → CPU

### 보조기억장치 (Secondary Storage)
📍 SSD / HDD
⚡ 속도: 가장 느림
📦 용량: 매우 큼 (수백 GB~TB)
❗ 비휘발성
🎯 영구 저장
프로그램, 사진, 영상 등 저장

## 데이터 이동 흐름
앱 실행 시 흐름:
SSD에 저장된 프로그램
→ RAM으로 로드
→ CPU가 필요한 데이터 캐시에 복사
→ 실제 연산은 레지스터에서 수행

## 종류가 여러가지인 이유
### 속도 차이 때문
CPU는 매우 빠르지만,
RAM은 상대적으로 느리다.
→ 중간에 캐시를 둬서 병목 완화

### 비용 문제
레지스터를 크게 만들면 → 가격 폭증
SSD를 레지스터처럼 빠르게 만들면 → 기술적으로 불가능 + 비용 문제
→ 속도, 가격, 용량의 균형 필요

자주 쓰는 데이터는 비싸고 빠른 곳에, 가끔 쓰는 데이터는 싸고 큰 곳에 보관하여 효율을 극대화

### 지역성 원리 (Locality Principle)
컴퓨터는 이런 특징이 있음:
시간 지역성: 방금 사용한 데이터는 또 사용될 확률 높음
공간 지역성: 특정 데이터 주변 데이터도 같이 사용됨
→ 그래서 캐시가 효과적임

### 휘발성 vs 비휘발성
작업을 할 때는 빠른 휘발성 메모리(RAM)가 유리하지만,
컴퓨터를 끄고도 데이터가 남아야 하므로 비휘발성 메모리(SSD)가 반드시 별도로 필요
