# 메모리가 고갈되면 일어나는 현상에 대해 설명해주세요.
## 1. 급격한 성능 저하 (Thrashing 현상)
가장 먼저 나타나는 증상은 속도 저하

원인: RAM 공간이 부족해지면 운영체제는 부족한 공간을 메우기 위해 하드디스크나 SSD의 일부를 RAM처럼 사용하는 **'가상 메모리'**를 과도하게 쓰기 시작

결과: SSD는 RAM보다 훨씬 느립니다. CPU가 실제 연산보다 데이터를 RAM과 SSD 사이에서 왔다 갔다 옮기는 데(Page In/Out) 더 많은 시간을 쓰게 되는데, 이를 **스래싱(Thrashing)**이라고 함. 마우스 커서가 버벅이거나 창 하나 띄우는 데 한참 걸리는 이유

## 2. 애플리케이션 강제 종료
메모리를 더 이상 쥐어짜낼 수 없을 때, 개별 앱들이 비정상적으로 작동하기 시작

에러 발생: 개발자라면 익숙한 OutOfMemory (OOM) 에러가 발생하며 실행 중이던 프로그램이 갑자기 꺼짐

데이터 손실: 작업 중이던 내용이 저장되지 않은 채 앱이 종료되어 데이터가 날아갈 위험

## 3. OS의 최후 수단: OOM Killer
시스템 전체가 뻗는 것을 막기 위해 운영체제(특히 리눅스나 안드로이드)는 특단의 조치를 취함

OOM Killer 작동: OS가 판단하기에 메모리를 많이 잡아먹거나 중요도가 낮은 프로세스를 임의로 골라 강제로 죽여버림

안드로이드 폰을 쓰다가 백그라운드에 둔 앱이 초기화되거나 다시 로딩되는 현상이 바로 이 때문

## 4. 시스템 프리징(Freezing) 및 재부팅
모든 관리 수단이 한계에 도달하면 아예 시스템이 반응하지 않게 됨

커널 패닉 / 블루스크린: 운영체제의 핵심 기능을 수행할 메모리조차 남지 않으면 윈도우는 **Blue Screen of Death (BSOD)**를 띄우고, 다른 OS들도 시스템 보호를 위해 스스로 재부팅을 시도

## 요약
1단계: 경고	전체적인 버벅임, 반응 속도 저하	스래싱(가상 메모리 과다 사용)
2단계: 장애	특정 앱의 강제 종료, 에러 메시지	메모리 할당 실패 (OOM)
3단계: 개입	백그라운드 앱 자동 종료	OS의 OOM Killer 작동
4단계: 마비	시스템 멈춤(Freezing), 블루스크린	커널 가용 메모리 고갈
