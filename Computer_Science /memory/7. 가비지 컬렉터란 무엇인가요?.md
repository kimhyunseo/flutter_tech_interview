# 가비지 컬렉터란 무엇인가요? 
프로그램에서 더 이상 사용되지 않는 객체를 자동으로 찾아 메모리에서 해제하는 시스템

## 가비지 컬렉션(GC)은 왜 필요한가요?
C 언어 같은 오래된 언어에서는 개발자가 메모리를 직접 할당하고(malloc)
사용이 끝나면 직접 해제(free)해야 했음
하지만 사람이 하는 일이다 보니 실수가 생길 수 있음

메모리 누수(Memory Leak): 다 쓴 메모리를 해제하지 않아 메모리가 꽉 차는 현상
허상 포인터(Dangling Pointer): 이미 해제된 메모리 주소를 가리키고 있는 현상

가비지 컬렉터는 이러한 번거로움과 위험을 대신 짊어지고, 개발자가 비즈니스 로직에만 집중할 수 있게 도와줌

## GC는 어떻게 동작하나?
> 핵심 개념은 참조(Reference)

GC는 이렇게 판단:
어떤 객체가
어디에서도 참조되지 않으면
그 객체는 “쓰레기”로 판단
그리고 메모리에서 제거

## 대표적인 GC 방식
### ① Mark and Sweep
사용 중인 객체 표시 (Mark)
표시 안 된 객체 제거 (Sweep)

### ② Generational GC (세대별 GC)
대부분의 현대 언어(Java, Python, Dart 등)는 효율을 높이기 위해 메모리를 '세대'별로 나눔
**"새로 만들어진 객체는 금방 죽고, 오래 살아남은 객체는 앞으로도 오래 살 가능성이 높다"**는 통계적 사실 때문

Young Generation (신세대): 새로 생성된 객체들이 머무는 곳. 이곳은 청소가 아주 자주 일어나며, 매우 빠름

Old Generation (구세대): 여러 번의 청소 작업에도 살아남은 '질긴' 객체들이 이동하는 곳. 여기는 덩치가 크기 때문에 청소하는 데 시간이 더 걸리고 덜 자주 일어남

## 가비지 컬렉션의 단점
Stop-the-world: GC가 메모리를 청소하는 동안 프로그램의 실행이 아주 잠깐 멈추는 현상

오버헤드: GC 자체가 CPU 자원을 사용하기 때문에 프로그램 성능에 영향을 줄 수 있음
