# Dart에서 가비지 컬렉터는 어떤 방식으로 작동하나요?
Dart는 세대별(Generational) + Mark-Sweep 기반 가비지 컬렉터를 사용

## 왜 세대별 GC를 쓰는가?
Dart는 이런 가정을 함:
🔥 대부분의 객체는 금방 죽는다

Flutter에서:
위젯
빌드 중 생성되는 객체
임시 리스트
→ 짧은 생명주기
그래서 메모리를 두 영역으로 나눠 관리

## Dart의 메모리 구조
### ① Young Generation (New Space)
새로 생성된 객체 저장
자주 GC 실행
빠른 수거
대부분 객체는 여기서 생성되고 여기서 사라짐

### ② Old Generation (Old Space)
오래 살아남은 객체 저장
GC 빈도 낮음
크기 큼
여기까지 살아남으면 “장수 객체”

## 실제 동작 방식
### Young 영역 → Scavenger (복사 방식)
Young 영역 GC는 빠른 복사 기반 방식 사용
살아있는 객체만 다른 공간으로 복사
나머지는 자동 제거
👉 매우 빠름
👉 Flutter에서 프레임 유지에 중요

### Old 영역 → Mark-Sweep
Old 영역은:
Mark (사용 중 표시)
Sweep (안 쓰는 객체 제거)
→ 상대적으로 느림

## 언제 GC가 실행되나?
Young 공간이 가득 찼을 때
메모리 임계치 도달
런타임 판단

Flutter 앱에서:
객체 과도 생성
rebuild 과다
→ Young GC 자주 발생

## GC와 Flutter 성능 관계
GC가 실행되면:
잠깐 멈춤 (Stop-the-world)
프레임 드랍 가능
그래서 중요한 것:
const 사용
객체 재사용
불필요한 rebuild 방지
