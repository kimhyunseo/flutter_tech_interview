# 캐시(cache) 메모리를 왜 사용하는지, CPU의 적중률을 높이기 위해 어떤 원리를 사용하는지에 대해 설명해주세요.
## 캐시 메모리를 왜 사용하는가?
### 핵심 이유: CPU와 RAM의 속도 차이
CPU는 매우 빠름 (ns 단위)
RAM은 상대적으로 느림
SSD는 더 느림
만약 CPU가 매번 RAM까지 가서 데이터를 가져오면?
👉 CPU가 기다리느라 놀게 됨 (병목 발생)
그래서 등장한 게 **캐시(Cache)**

### 🔥 캐시의 역할
CPU가 자주 사용하는 데이터를 미리 저장해두는 고속 메모리
CPU 근처에 있음
SRAM 사용 (RAM은 DRAM)
용량은 작지만 매우 빠름

### 📌 한 줄 정리
캐시는 CPU와 RAM 사이의 속도 차이를 줄여 병목을 완화하기 위해 사용된다.

## CPU 적중률(Hit Rate)을 높이기 위한 원리
### 핵심 개념: 지역성(Locality)
캐시가 효과적인 이유는
프로그램이 특정 패턴으로 메모리를 사용하기 때문

### 시간 지역성 (Temporal Locality)
방금 사용한 데이터는 곧 다시 사용될 가능성이 높다.
예:
```dart
for (int i = 0; i < 1000; i++) {
    sum += arr[i];
}
```
반복문 안에서 같은 변수, 같은 메모리를 계속 사용
👉 최근 사용 데이터를 캐시에 유지

### 공간 지역성 (Spatial Locality)
특정 데이터를 사용하면, 그 근처 데이터도 사용할 가능성이 높다.
예:
```dart
for (int i = 0; i < 1000; i++) {
    sum += arr[i];
}
```
배열을 순차적으로 접근
인접 메모리를 함께 가져옴
👉 캐시는 한 번에 한 블록(캐시 라인) 단위로 가져옴

## 캐시 적중(Hit) / 실패(Miss)
Hit: 캐시에 데이터 있음 → 빠름
Miss: 없음 → RAM까지 감 → 느림
적중률 = Hit / 전체 접근

## 적중률을 높이기 위한 전략
### 블록 단위 저장
공간 지역성 활용
→ 캐시 라인 단위로 저장

### LRU 같은 교체 알고리즘
오래 안 쓴 데이터 제거
최근 사용 데이터 유지

대표 알고리즘:
LRU (최근 사용 최소)
LFU (최빈 사용 최소)

### 멀티 레벨 캐시 (L1, L2, L3)
L1: 가장 빠름, 가장 작음
L2: 중간
L3: 크지만 느림
계층 구조로 적중 확률 증가

## 정리
캐시 메모리는 CPU와 RAM 사이의 속도 차이를 줄이기 위해 사용됩니다.
CPU는 매우 빠르지만 RAM은 상대적으로 느리기 때문에,
자주 사용하는 데이터를 캐시에 저장하여 접근 시간을 줄입니다.
캐시는 지역성 원리를 기반으로 동작합니다.
시간 지역성은 최근 사용한 데이터를 다시 사용할 가능성이 높다는 원리이고,
공간 지역성은 특정 데이터 주변 데이터도 함께 사용될 가능성이 높다는 원리입니다.
이를 통해 캐시 적중률을 높이고 전체 시스템 성능을 향상시킵니다.
