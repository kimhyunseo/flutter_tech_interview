# 추상화란?
복잡한 내부 구현 로직은 숨기고, 사용자에게 필요한 핵심적인 기능(인터페이스)만을 드러내는 것
**"무엇(What)을 하는지는 알려주되, 어떻게(How) 하는지는 감추는 것"**

## 추상화를 하는 이유
- 복잡성 감소: 내부의 지저분한 코드를 몰라도 기능을 사용할 수 있음
- 유연성 및 확장성: 내부 구현(How)을 바꿔도 이를 사용하는 쪽(What)은 영향을 받지 않음
- 공통 분모 추출: 여러 객체(야구장, 축구장, 농구장)의 공통점(경기장 이름, 위치 정보)만 뽑아내어 관리하기 편하게 만듦

## 코드 예시 (Dart)
```dart
abstract class Animal {
  void makeSound();
}
```

“동물은 소리를 낸다”라는 개념만 정의
어떻게 소리를 내는지는 각 클래스가 구현

```dart
class Dog extends Animal {
  @override
  void makeSound() {
    print("멍멍");
  }
}
```
👉 공통 개념만 정의하고
👉 구체적인 구현은 숨김
