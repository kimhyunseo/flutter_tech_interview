# 객체지향이란 무엇인지 설명하세요.
## 객체 지향이란?
객체를 중심으로 세상을 바라보고 문제를 해결하려는 사고 방식 또는 패러다임
현실 세계를 객체 단위로 모델링해서 생각하는 방식

예를 들어:
사람 → 이름, 나이 / 말한다, 걷는다
자동차 → 속도, 색상 / 달린다, 멈춘다

👉 이렇게 **데이터(상태) + 행동(기능)**을 함께 묶어서 생각하는 사고 방식이 객체 지향

## 객체지향 프로그래밍(OOP)이란?
현실 세계의 '객체'들을 프로그램으로 표현하는 방식
현실 세계의 개념을 객체(Object) 단위로 모델링하여, 객체 간의 상호작용으로 프로그램을 구성하는 프로그래밍 패러다임

객체 지향 개념을 기반으로 작성된 프로그램
즉, 객체를 생성하고 객체 간의 상호작용으로 시스템이 동작하도록 만든 프로그램

**객체란?**
속성 (데이터, 상태)
행위 (메서드, 기능)
를 함께 가지는 단위

## 객체지향의 핵심 특징 4가지

### 1. 캡슐화 (Encapsulation)
개념: 데이터(속성)와 해당 데이터를 처리하는 메서드를 하나의 객체로 묶고,
외부에서 직접 접근하지 못하도록 보호하는 것
목적: 데이터 보호와 독립성 확보. 내부가 어떻게 바뀌든 외부에선 정해진 방법으로만 소통하면 됩니다.

### 2. 상속 (Inheritance)
개념: 이미 만들어진 클래스의 특성을 그대로 물려받아 새로운 클래스를 만드는 것
목적: 코드 재사용성과 계층 구조 형성
비유: '스마트폰'이라는 부모 클래스를 상속받아 '아이폰'과 '갤럭시'라는 자식 클래스를 만드는 것

### 3. 추상화 (Abstraction)

개념: 복잡한 내부 로직은 숨기고, 사용자에게 필요한 핵심적인 기능만을 인터페이스로 드러내는 것
목적: 복잡성 감소 및 설계의 유연성. "무엇(What)을 하는가"에 집중

### 4. 다형성 (Polymorphism)
개념: 하나의 인터페이스나 형태가 상황에 따라 여러 가지 방식으로 동작하는 능력
목적: 코드의 유연한 확장. 구체적인 타입에 의존하지 않는 코드를 짤 수 있습니다.
비유: '키보드의 Enter 키'. 채팅창에서는 '전송'이지만, 문서 도구에서는 '줄바꿈'으로 동작하는 것

---

| 특징 | 핵심 키워드 | 주요 장점 |
| --- | --- | --- |
| 캡슐화 | 정보 은닉, 보호 | 보안성, 독립성 유지 |
| 상속 | 재사용, 계층 구조 | 코드 중복 제거, 개발 속도 향상 |
| 추상화 | 단순화, 설계도 | 복잡성 감소, 설계의 유연성 |
| 다형성 | 재정의, 유연함 | 확장성, 유지보수 용이 |
