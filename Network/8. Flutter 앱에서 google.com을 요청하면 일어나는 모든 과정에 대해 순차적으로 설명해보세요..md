# Flutter 앱에서 “[https://google.com](https://google.com/)”을 요청하면 일어나는 모든 과정에 대해 순차적으로 설명해보세요.


## 1단계: Dart 코드의 실행 (Application Layer)
Flutter 앱에서 http.get()이나 dio.get() 같은 메서드가 호출
이 요청은 Dart Isolate 내에서 비동기(Future)로 처리
요청 헤더(User-Agent, Accept 등)가 생성되고, OS의 네트워크 스택으로 전달

## 2단계: DNS 조회 (Domain Name Resolution)
컴퓨터는 google.com이 어디인지 모름. 숫자로 된 주소(IP)가 필요
앱은 시스템 DNS 캐시를 확인하고, 없으면 DNS 서버에 물어봄
"https://www.google.com/url?sa=E&source=gmail&q=google.com의 IP 주소가 뭐야?" -> "142.250.206.238 이야!"라고 답변을 받아옴

## 3단계: TCP 연결 (Transport Layer)
주소를 알았으니 통로를 뚫어야 함. TCP 3-Way Handshake가 발생

Client: "연결할 수 있을까요? (SYN)"

Server: "네, 가능합니다. 당신도 준비됐나요? (SYN-ACK)"

Client: "네, 준비됐습니다. 연결할게요! (ACK)"

## 4단계: TLS/SSL 핸드쉐이크 (Security Layer)
HTTPS이기 때문에 가장 중요한 보안 연결 과정이 추가됨

서버가 자신의 인증서와 공개키를 보냄

앱(브라우저/OS)은 이 인증서가 믿을만한지 확인

서로만 아는 대칭키를 안전하게 생성하여 이후 모든 데이터를 암호화

## 5단계: HTTP 요청 전송 (HTTP Request)
이제 안전한 통로로 진짜 하고 싶은 말을 보냄

GET / HTTP/1.1

Host: google.com

암호화된 이 메시지가 패킷 단위로 쪼개져 구글 서버로 날아감

## 6단계: 구글 서버의 처리 (Server Side)
구글의 거대한 인프라가 작동

로드 밸런서가 수많은 서버 중 하나로 요청을 배정

웹 서버가 요청을 해석하고, 필요한 데이터(검색 결과 등)를 처리한 뒤 HTML이나 JSON 응답을 만듦

## 7단계: HTTP 응답 수신 (HTTP Response)
서버가 앱으로 답변을 보냄

HTTP/1.1 200 OK

데이터 본문(Body)에 구글 페이지의 HTML이나 데이터가 담겨 있음

데이터가 모두 도착하면 TCP 연결은 효율을 위해 잠시 유지되거나 끊어짐

## 8단계: Flutter UI 업데이트 (Rendering)
다시 앱으로 돌아옴

역직렬화: 받은 데이터가 JSON이라면 Dart 객체로 변환

State 업데이트: setState()나 상태 관리 도구(Provider, Bloc 등)를 통해 데이터를 화면에 반영

다시 그리기: Flutter의 위젯 트리(Widget Tree)가 갱신되며 사용자의 화면에 구글 페이지가 나타남

    ↓
Flutter UI 업데이트
