# Oauth2.0 이란 무엇인가요?
OAuth 2.0은 권한 부여(Authorization) 프레임워크
사용자의 비밀번호를 직접 전달하지 않고, 타사 애플리케이션이 사용자의 데이터에 접근하도록 허용하는 안전한 방법을 제공
주로 웹, 모바일 앱, API에서 사용
핵심: “누가 어떤 권한으로 무엇에 접근할 수 있는지를 안전하게 관리하는 규칙”

## Oauth2.0의 구성 요소
| 역할                           | 설명                                       |
| ---------------------------- | ---------------------------------------- |
| **Resource Owner (사용자)**     | 자신의 데이터에 접근 권한을 가진 사람                    |
| **Client (앱)**               | 사용자 데이터를 사용하려는 애플리케이션                    |
| **Authorization Server**     | 사용자를 인증하고, **Access Token**을 발급하는 서버     |
| **Resource Server (API 서버)** | 실제 데이터(API)를 가진 서버, Access Token으로 요청 검증 |

## Oauth2.0의 동작 과정
동작 흐름 (Authorization Code Grant 기준)

1. 사용자 인증 요청
앱(Client)이 Authorization Server에 권한 요청
URL로 사용자 로그인 페이지 이동

2. 사용자 동의
사용자가 “이 앱이 내 데이터 접근하는 것을 허용” 선택

3. Authorization Code 발급
서버가 일회용 코드(Code) 발급 후 앱에 전달

4. Access Token 발급
앱이 Authorization Code + Client Secret으로 Access Token 요청
서버가 Access Token 발급 → 앱이 저장

5. API 호출
앱이 Access Token을 포함해 Resource Server에 요청
Resource Server가 토큰 검증 후 데이터 반환

## 특징
- 비밀번호 미전달: 앱은 사용자의 비밀번호를 알 필요 없음
- 사용자 편의성(UX): 새로 회원가입할 필요 없이 기존 계정으로 1초 만에 시작할 수 있습니다.
- 권한 제어: "이메일은 봐도 되지만, 구글 드라이브 파일은 건드리지 마!"처럼 권한 범위를 아주 세밀하게 조절할 수 있습니다.
- 토큰 기반 접근: Access Token으로만 인증 → 만료 가능, 회수 가능
- 다양한 인증 방식 지원: Authorization Code, Client Credentials, Implicit, Resource Owner Password 등
