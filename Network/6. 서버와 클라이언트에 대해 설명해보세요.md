# 서버와 클라이언트에 대해 설명해보세요.
**서버(Server)** 와 **클라이언트(Client)** 는 네트워크를 통해 서비스를 주고받는 두 주체 사이의 역할 관계를 의미

## 1. **클라이언트 (Client)**: 요청하는 주체

클라이언트는 서비스를 사용하는 사용자(User) 측의 단말기나 소프트웨어를 의미

역할: 사용자로부터 입력을 받아 서버에 **요청(Request)**을 보내고, 서버로부터 받은 **응답(Response)**을 화면에 보여줍니다.

주요 형태:

- 웹 브라우저: Chrome, Safari, Edge 등
- 모바일 앱: 스마트폰에서 실행되는 각종 애플리케이션
- 데스크톱 앱: PC용 소프트웨어

특징: 사용자 인터페이스(UI)와 사용자 경험(UX)에 집중합니다.

## 2. **서버 (Server)**: 응답하는 주체

서버는 클라이언트의 요청을 받아 처리하고 그 결과를 제공하는 제공자입니다.

역할: 클라이언트의 요청이 유효한지 확인하고, 필요한 데이터(DB 조회 등)를 처리한 뒤 다시 클라이언트에게 **응답(Response)**을 보냅니다.

주요 종류:

- 웹 서버: 웹 페이지(HTML, CSS)를 전달함
- API 서버: 데이터(JSON 등)를 전문적으로 전달함
- DB 서버: 데이터를 저장하고 관리함

특징: 24시간 내내 켜져 있어야 하며, 수많은 클라이언트의 요청을 동시에 처리할 수 있는 성능이 필요합니다.

## 역할 분리의 이유

1. 관심사 분리 (Separation of Concerns)
클라이언트 → 화면/UI
서버 → 데이터 처리 / 보안

2. 확장성
서버만 증설 가능
클라이언트 업데이트 없이 서버 로직 변경 가능
