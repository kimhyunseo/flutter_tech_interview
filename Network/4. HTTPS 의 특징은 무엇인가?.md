# HTTPS 의 특징은 무엇인가?
HTTPS는 HTTP에 SSL/TLS 암호화 계층을 추가한 보안 프로토콜

## HTTPS의 주요 특징
### 1. 데이터 암호화 (Encryption)
클라이언트와 서버가 주고받는 데이터를 암호화
중간에 누군가 데이터를 가로채더라도(스니핑), 암호화되어 있어 내용을 알 수 없음

공개키(Asymmetric)와 대칭키(Symmetric) 암호화 방식을 혼합해서 사용

### 2. 서버 인증 (Authentication)
내가 접속한 사이트가 진짜 그 사이트가 맞는지 확인
CA(인증 기관)라는 신뢰할 수 있는 제3자가 발급한 '디지털 인증서'를 통해 서버의 신원을 보증
피싱 사이트(가짜 사이트)로 유인되어 개인정보를 뺏기는 일을 막아줌

### 3. 데이터 무결성 보장 (Integrity)
데이터가 전송되는 도중에 해커가 내용을 조작하는 것을 방지
전송된 데이터가 원본과 일치하는지 검증하는 과정을 거침
만약 1이라도 수정되었다면 바로 알아차릴 수 있음

### 4. 기본 포트 번호
HTTP: 80
HTTPS: 443

## HTTPS 동작 원리
브라우저와 서버가 처음 만나서 "우리 이제부터 이렇게 암호화하자!"라고 약속하는 과정을 SSL/TLS 핸드쉐이크라고 함

- 인사: 클라이언트가 서버에 접속하면 서로 지원하는 암호화 방식을 확인
- 신분증 확인: 서버가 클라이언트에게 자신의 인증서를 보냄
- 암호 키 생성: 클라이언트는 인증서 안에 들어있는 '공개키'를 이용해, 앞으로 통신에 사용할 '비밀키'를 안전하게 만들어 전달
- 암호화 통신: 이제부터는 둘만 아는 그 '비밀키'로 모든 데이터를 암호화해서 주고받음

## HTTP vs HTTPS 비교
| 항목  | HTTP | HTTPS     |
| --- | ---- | --------- |
| 암호화 | ❌ 없음 | ✅ TLS 암호화 |
| 보안성 | 낮음   | 높음        |
| 포트  | 80   | 443       |
| 인증서 | 없음   | 필요        |
| SEO | 낮음   | 검색엔진 우대   |

## HTTPS의 단점
암호화 과정으로 인한 약간의 성능 오버헤드
인증서 비용 (요즘은 무료도 많음)
설정 복잡성
하지만 현재는 거의 모든 서비스가 HTTPS 사용

## 면접용 정리 답변
HTTPS는 HTTP에 SSL/TLS 암호화 계층을 추가한 보안 프로토콜입니다.
데이터 암호화, 서버 인증, 데이터 무결성 보장을 특징으로 합니다.
공개키와 대칭키 암호화를 함께 사용하여 보안을 확보하며, 기본 포트는 443번입니다.
HTTP보다 보안성이 높아 현재 대부분의 웹 서비스에서 사용됩니다.

"HTTPS는 HTTP에 SSL/TLS 계층을 추가하여 기밀성, 무결성, 인증을 보장하는 프로토콜입니다.
단순히 데이터를 암호화하는 것을 넘어, CA를 통한 서버 인증으로 피싱을 방지하고 데이터 위변조를 막습니다. 현대 웹에서 HTTPS는 선택이 아닌 필수이며, 구글과 같은 검색 엔진에서도 보안이 적용된 사이트에 가산점을 주기 때문에 마케팅 및 SEO 측면에서도 매우 중요합니다."
